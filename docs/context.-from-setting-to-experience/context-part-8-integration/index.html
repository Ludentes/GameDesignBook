Part 8: Integration and Coherence

The Coherence Requirement

You've made your context decisions:

  • ✓ Chosen genre from aesthetic goals
  • ✓ Separated genre (load-bearing) from dressing (cosmetic)
  • ✓ Defined your perspective
  • ✓ Made Build/Suggest/Omit decisions for each element
  • ✓ Created your context design document

Now the critical question: Does everything work together?

Coherence means all elements point in the same direction toward the same aesthetic goal. When coherent, players intuitively understand what the game wants them to do. When incoherent, players receive contradictory signals and feel confused about what matters.

This chapter shows you how to achieve and validate coherence.

What Must Align

Context doesn't exist in isolation. It must integrate with every other system in your game:

1. Genre and Aesthetic

The foundation alignment:

  • Genre must naturally create your primary aesthetic
  • Secondary aesthetics must be compatible with genre
  • Dressing must reinforce, not contradict

Example - Mystery Investigation:

  • Primary aesthetic: Discovery (investigation and deduction)
  • Secondary aesthetic: Narrative (story through investigation)
  • Genre: Mystery investigation ✓ (serves both)
  • Dressing: Victorian London ✓ (reinforces atmosphere)

Anti-example - Space Coffee Failure:

  • Primary aesthetic: Discovery
  • Genre: Generic open-world exploration ✗
  • Dressing: Sci-fi planets
  • Problem: 18 quintillion planets, none memorable (no Discovery)

2. Perspective and Content

Everything must serve perspective:

  • Build decisions must reinforce perspective
  • Suggest decisions must maintain mystery while supporting perspective
  • Omit decisions must preserve what perspective needs hidden

Example - Horror:

  • Perspective: "Unknowable threat"
  • Build: Attack patterns (need to survive) ✓
  • Suggest: Physical appearance (glimpses) ✓
  • Omit: Motivation, sympathetic framing ✓
  • Result: Threat stays unknowable

Anti-example - Perspective Break:

  • Perspective: "Unknowable threat"
  • Add: Alien doing comedy pratfalls ✗
  • Result: Horror becomes comedy, perspective broken

3. Context and Mechanics

From Chapter 1 (Section 3: Five Components): Mechanics must fit context

  • Challenge mechanics in Challenge context
  • Investigation mechanics in Mystery context
  • No combat in Cozy context

Example - Blades in the Dark:

  • Context: Victorian criminals, desperate heists
  • Mechanics: Flashbacks (heist genre), Stress (desperation), Heat (consequences)
  • Result: Mechanics emerge naturally from context ✓

Anti-example - D&D 4e:

  • Context: D&D fantasy adventure
  • Mechanics: Tactical board game powers
  • Result: Mechanics contradict context ✗

4. Context and Resources

From Chapter 2: Resources must make sense in context

  • What resources exist in this world?
  • How does context explain resource recovery?
  • Do resource types serve aesthetic?

Example - Horror:

  • Context: Survival against alien threat
  • Resources: Health (physical vulnerability), Ammunition (scarcity), Hiding spots (limited safety)
  • Recovery: Slow/none (maintains tension)
  • Result: Resources reinforce horror ✓

Example - Cozy:

  • Context: Peaceful library restoration
  • Resources: Energy (gentle depletion), Materials (abundant), Contentment (grows naturally)
  • Recovery: Fast and forgiving (no pressure)
  • Result: Resources reinforce cozy aesthetic ✓

Anti-example - Andromeda:

  • Context narrative: "Desperate colonization, survival at stake"
  • Resources actual: Abundant, fast recovery, no real scarcity
  • Result: Context says desperate, mechanics say safe ✗

5. Context and Balance Paradigm

From Chapter 4: Paradigm must fit genre

  • Framework paradigm: Narrative flexibility (mysteries, heists, social)
  • Consistency paradigm: Pattern learning (challenges, boss fights)
  • Adaptive paradigm: Dynamic response (cooperation, discovery)

Example - Mystery Investigation:

  • Genre: Mystery investigation
  • Paradigm: Framework (need flexibility for unpredictable investigation approaches) ✓
  • Alternative: Consistency (fixed clues, rigid patterns) ✗ would break investigation

Example - Boss Fight Challenge:

  • Genre: Tactical boss combat
  • Paradigm: Consistency (learnable patterns) ✓
  • Alternative: Adaptive (boss changes every time) ✗ would prevent mastery

Example - Horror Survival:

  • Genre: Horror with unknowable threat
  • Paradigm: Adaptive (threat responds to player behavior, unpredictable) ✓
  • Alternative: Consistency (learnable patterns) ✗ would reduce fear

6. Context and Cognitive Load

From Chapter 3: Load budget must match aesthetic

  • Challenge/Mastery: High load acceptable (optimization expected)
  • Fellowship: Low mechanical load (brainspace for socializing)
  • Submission: Low load required (relaxation goal)
  • Discovery: Medium load (investigation thinking, not calculation)

Example - Cozy Game:

  • Context: Relaxing restoration
  • Load: Simple mechanics, clear systems, minimal tracking ✓
  • Result: Can relax (serves Submission)

Anti-example - Cozy Game with Complexity:

  • Context: Relaxing farming
  • Load: 20 resource types, complex optimization, hidden mechanics ✗
  • Result: Stressful not relaxing (breaks aesthetic)

7. Context and Dynamics

From Part 7: Dynamics must serve aesthetic

  • Context creates expected behaviors
  • Behaviors must match intended aesthetic
  • Genre expectations drive dynamics

Example - Mystery:

  • Context: Solvable murder mystery
  • Expected dynamics: Investigate, question NPCs, connect clues
  • Mechanics support: Investigation mechanics, NPC dialogue, clue tracking ✓
  • Result: Players naturally investigate

Anti-example - Mystery with Combat Focus:

  • Context: Murder mystery
  • Mechanics: D&D combat-focused
  • Actual dynamics: Players kill suspects, loot bodies for clues ✗
  • Result: Violence dynamics, not investigation dynamics

The Key Insight

SAME MECHANICS. OPPOSITE FEELINGS. PERFECT COHERENCE IN BOTH.

This proves coherence is achievable through:

  1. Clear aesthetic goals
  2. Context that serves those goals
  3. Parameters tuned to aesthetic
  4. Everything pointing same direction

Not about complexity of mechanics. About alignment of all elements.

Incoherence: What Goes Wrong

Case Study 1: Mass Effect Andromeda

What they said:

  • "Desperate colonization mission"
  • "Survival against odds"
  • "Pathfinder finding new home"

What the mechanics actually were:

  • Abundant resources
  • Fast recovery
  • Safe exploration
  • No real stakes

The incoherence:

ElementWhat It SaidWhat It Did
Narrative"Desperate survival""Abundant and safe"
Context"Hostile unknown galaxy""Generic collection zones"
ResourcesShould be scarceActually abundant
DeathShould matterInstant respawn
ExplorationShould be discoveryWas checklist completion

Result: Players didn't believe the stakes. Context promised tension, mechanics delivered theme park.

What should have aligned:

  • IF narrative says desperate → Resources scarce, recovery slow, stakes high
  • OR IF mechanics say abundant → Narrative should say "well-supplied colonization"
  • Pick one. They picked neither. Incoherent.

Case Study 2: D&D 4th Edition

What they said:

  • "D&D fantasy adventure"
  • "Traditional dungeon crawling"
  • "Heroic epic fantasy"

What the mechanics actually were:

  • Tactical board game on mandatory grid
  • Powers system (like MMO hotbars)
  • Highly balanced but mechanically focused
  • No mechanical support for non-combat

The incoherence:

ElementD&D Context4e Mechanics
Identity"Theater of the mind" roleplayingMandatory tactical grid
ClassesDistinct fantasy archetypesAll have same power structure
MagicWondrous and dangerousBalanced resource like martial powers
CombatPart of adventurePrimary focus of rules
SocialImportant in D&DNo mechanical support

Result: Technically superior balance, but lost D&D's identity. Context said "D&D," mechanics said "tactical board game."

What should have aligned:

  • IF keeping D&D context → Mechanics should support traditional play
  • OR IF using tactical mechanics → Should have rebranded (not called it D&D)
  • Can't have D&D context + non-D&D mechanics. Incoherent.

How to Achieve Coherence

Step 1: Start with Aesthetic (Chapter 1, Section 2)

Foundation of coherence:

  • Primary aesthetic clearly defined
  • Secondary aesthetics compatible
  • Aesthetic goals written down and referenced constantly

Example:

  • Primary: Discovery (investigation satisfaction)
  • Secondary: Narrative (story through investigation)
  • Written goal: "Players solve mysteries through deduction and feel satisfied when connections click"

Step 2: Choose Genre That Creates Aesthetic (Part 2)

Genre must naturally support aesthetic:

  • Use decision tree
  • Filter by constraints
  • Validate genre actually creates aesthetic

Example:

  • Aesthetic: Discovery + Narrative
  • Genre candidates: Mystery investigation, Archaeological discovery
  • Choice: Mystery investigation (serves both well)
  • Validation: Mystery naturally creates investigation dynamics ✓

Step 3: Establish Perspective (Part 4)

Clear perspective statement:

  • One sentence: What feeling/framing creates aesthetic
  • All elements must serve this

Example:

  • Perspective: "Solvable puzzle where deduction yields satisfying answers"
  • Test each element: Does this serve solvable deduction?

Step 4: Make Build/Suggest/Omit Decisions (Part 5)

Decisions must align with aesthetic:

  • Discovery aesthetic → High Suggest (fragments to connect)
  • Horror aesthetic → High Omit (fear of unknown)
  • Challenge aesthetic → High Build (clarity for fairness)

Example - Mystery:

  • Build: Core clues (need to solve)
  • Suggest: Connections between clues (deduction is the game)
  • Omit: Solution (until earned)
  • Result: Serves Discovery aesthetic ✓

Step 5: Choose Resources That Fit Context (Chapter 2)

Resources must make narrative sense:

  • What resources exist in this world?
  • How does context explain them?
  • Do recovery patterns serve aesthetic?

Example - Horror:

  • Resource: Health (physical vulnerability in universe with deadly threat)
  • Recovery: None or slow (maintains tension)
  • Explains why: Limited medical supplies, isolated location
  • Result: Resources serve context and aesthetic ✓

Step 6: Choose Balance Paradigm That Fits Genre (Chapter 4)

Paradigm must match genre needs:

  • Framework: Narrative flexibility (unpredictable player approaches)
  • Consistency: Pattern learning (tactical challenges)
  • Adaptive: Dynamic response (cooperative or discovery)

Example - Mystery:

  • Genre: Investigation (unpredictable approaches)
  • Paradigm: Framework (GM/LLM handles creative investigation) ✓
  • Wrong choice: Consistency (rigid clue finding) ✗

Step 7: Set Cognitive Load Budget (Chapter 3)

Load must match aesthetic:

  • Submission: Low load required
  • Mastery: High load acceptable
  • Fellowship: Low mechanical load (brainspace for social)

Example - Cozy:

  • Aesthetic: Submission (relaxation)
  • Load: Low (simple systems, clear feedback) ✓
  • Wrong choice: High (complex optimization) ✗

Step 8: Validate Dynamics (Part 7)

Expected behaviors must match aesthetic:

  • What should players DO?
  • Does context create those behaviors?
  • Do mechanics support those behaviors?

Example - Mystery:

  • Expected: Players investigate, question, deduce
  • Context: Mystery genre (creates investigation expectation) ✓
  • Mechanics: Investigation tools, NPC dialogue, clue tracking ✓
  • Result: Dynamics align ✓

Step 9: Check for Contradictions

Look for elements that fight each other:

  • Context says one thing, mechanics say another?
  • Resources contradict narrative?
  • Cognitive load wrong for aesthetic?
  • Dynamics don't match intention?

Common contradictions:

  • Desperate narrative + abundant resources (Andromeda)
  • D&D context + tactical board game mechanics (4e)
  • Cozy aesthetic + high pressure mechanics
  • Mystery genre + combat-focused mechanics

Fix contradictions by:

  • Changing context to match mechanics, OR
  • Changing mechanics to match context, OR
  • Accepting incoherence (not recommended)

The Coherence Validation Checklist

Use this before finalizing design:

Foundation Alignment

□ Aesthetic clearly defined (primary + secondary) □ Genre serves aesthetic (naturally creates intended experience) □ Dressing reinforces genre (doesn't contradict) □ Perspective statement clear (one sentence framing)

Content Alignment

□ Build decisions serve aesthetic (right amount of detail) □ Suggest decisions serve aesthetic (productive mystery) □ Omit decisions serve aesthetic (strategic absence) □ No perspective breaks (every element maintains feeling)

System Integration

□ Resources fit context (make narrative sense) □ Resource recovery serves aesthetic (tension vs comfort) □ Balance paradigm fits genre (Framework/Consistency/Adaptive match) □ Cognitive load matches aesthetic (right amount of thinking)

Mechanical Alignment

□ Mechanics fit context (emerge naturally from world) □ Mechanics serve aesthetic (create intended experience) □ Parameters tuned to aesthetic (scarcity vs abundance) □ No mechanical contradictions (combat in cozy game?)

Dynamic Alignment

□ Expected behaviors defined (what should players do?) □ Context creates behaviors (genre expectations drive dynamics) □ Mechanics support behaviors (tools for intended actions) □ Dynamics serve aesthetic (behaviors create experience)

No Contradictions

□ Context doesn't contradict mechanics (Andromeda problem) □ Narrative doesn't contradict resources (desperate but abundant?) □ Perspective maintained throughout (no comedy in horror) □ All elements point same direction (coherent whole)

If you can check all boxes → Coherent design If multiple boxes unchecked → Identify contradictions and fix

Integration Points: Practical Examples

Example 1: Horror Survival Game

Foundation:

  • Primary aesthetic: Tension (high-stakes pressure)
  • Secondary aesthetic: Challenge (overcoming difficulty)
  • Genre: Horror survival
  • Perspective: "Unknowable threat hunting vulnerable humans"

Integration:

Context decisions:

  • Build: Threat behaviors, survival mechanics, safe zones
  • Suggest: Threat origins (fragments), alien lifecycle (hints)
  • Omit: Threat motivation, sympathetic framing

Resources (Chapter 2):

  • Health: No regeneration (permanent loss creates tension) ✓
  • Ammunition: Scarce (can't fight way out) ✓
  • Hiding places: Limited (must be strategic) ✓
  • All resources serve Tension aesthetic

Balance (Chapter 4):

  • Paradigm: Adaptive (threat responds to behavior, unpredictable) ✓
  • Serves Tension (can't fully predict) and Challenge (must adapt)

Cognitive Load (Chapter 3):

  • Load: Medium-High (tracking threat, managing resources, planning)
  • Acceptable for Tension aesthetic (pressure is point)

Dynamics:

  • Expected: Cautious movement, resource conservation, strategic hiding
  • Context creates: Fear of unknown (Omit decisions) → cautious behavior ✓
  • Mechanics support: Stealth mechanics, resource scarcity, limited saves ✓

Coherence check: ✓ Everything serves Tension aesthetic ✓ Resources are scarce (matches narrative) ✓ Threat is unknowable (maintains horror) ✓ Dynamics match intention (players act cautiously) ✓ No contradictions

Example 2: Cozy Library Restoration

Foundation:

  • Primary aesthetic: Submission (relaxation)
  • Secondary aesthetic: Discovery (pleasant finding)
  • Genre: Cozy restoration/organization
  • Perspective: "Safe discovery and gentle progress"

Integration:

Context decisions:

  • Build: Library systems, visitor requests, restoration tasks
  • Suggest: Library history (optional lore), book backstories
  • Omit: Threats, darkness, pressure, failure states

Resources (Chapter 2):

  • Energy: Abundant, fast recovery (no pressure) ✓
  • Materials: Plentiful (no scarcity stress) ✓
  • Time: Unlimited (no deadlines) ✓
  • All resources serve Submission aesthetic

Balance (Chapter 4):

  • Paradigm: Framework (total flexibility, no failure possible) ✓
  • Serves Submission (can't lose) and Discovery (free exploration)

Cognitive Load (Chapter 3):

  • Load: Low (simple systems, clear feedback, optional optimization) ✓
  • Serves Submission (can turn brain off if desired)

Dynamics:

  • Expected: Relaxed exploration, peaceful organization, casual play
  • Context creates: No threats (Omit) + gentle tasks → relaxed behavior ✓
  • Mechanics support: Simple sorting, forgiving systems, no time pressure ✓

Coherence check: ✓ Everything serves Submission aesthetic ✓ Resources are abundant (matches peaceful narrative) ✓ No threats or pressure (maintains cozy) ✓ Dynamics match intention (players relax) ✓ No contradictions

Example 3: Mystery Investigation

Foundation:

  • Primary aesthetic: Discovery (deduction satisfaction)
  • Secondary aesthetic: Narrative (story through investigation)
  • Genre: Mystery investigation
  • Perspective: "Solvable puzzle through deduction"

Integration:

Context decisions:

  • Build: Core clues (guaranteed findable), crime scene details, NPC alibis
  • Suggest: Motivations, relationships, hidden connections
  • Omit: Solution (until deduced), some red herrings initially plausible

Resources (Chapter 2):

  • Investigation points: Spend for optional depth clues ✓
  • Time: Pressure (killer may strike again) ✓
  • Reputation: Social resource for access ✓
  • Resources serve Discovery (investigation) and Narrative (consequences)

Balance (Chapter 4):

  • Paradigm: Framework (unpredictable investigation approaches need flexibility) ✓
  • Serves Discovery (can investigate creatively)

Cognitive Load (Chapter 3):

  • Load: Medium (tracking clues, forming theories, deduction)
  • Appropriate for Discovery aesthetic (investigation requires thinking)

Dynamics:

  • Expected: Active investigation, theory formation, connecting clues
  • Context creates: Mystery (Suggest clues) → investigation behavior ✓
  • Mechanics support: Clue tracking, NPC questioning, deduction tools ✓

Coherence check: ✓ Everything serves Discovery aesthetic ✓ Resources support investigation ✓ Mystery is solvable (Build core clues) ✓ Dynamics match intention (players investigate) ✓ No contradictions

Common Integration Failures

Failure 1: Wrong Resources for Context

Symptom:

  • Context says "desperate survival"
  • Resources abundant and fast-recovering

Example: Andromeda

  • Narrative: Desperate colonization
  • Resources: Abundant materials, fast health recovery, no real scarcity
  • Result: Players don't believe stakes

Fix:

  • Match resources to narrative (if desperate, make scarce)
  • OR change narrative to match mechanics (if abundant, say well-supplied)

Failure 2: Wrong Paradigm for Genre

Symptom:

  • Genre needs flexibility
  • Paradigm is rigid consistency

Example: Mystery with fixed investigation paths

  • Genre: Mystery (unpredictable player approaches)
  • Paradigm: Consistency (rigid clue sequence)
  • Result: Players frustrated when creative approaches don't work

Fix:

  • Use Framework paradigm for genres needing flexibility
  • Use Consistency paradigm for pattern-learning genres
  • Match paradigm to genre needs

Failure 3: Wrong Load for Aesthetic

Symptom:

  • Aesthetic needs relaxation
  • High cognitive load required

Example: Cozy game with complex optimization

  • Aesthetic: Submission (want to relax)
  • Load: High (20 resource types, complex calculations)
  • Result: Stressful not relaxing

Fix:

  • Simplify mechanics for Submission aesthetic
  • OR accept this serves Mastery not Submission
  • Match load to aesthetic

Failure 4: Contradictory Dynamics

Symptom:

  • Want investigation behavior
  • Mechanics reward violence

Example: Mystery in D&D

  • Want: Investigation, questioning, deduction
  • Mechanics reward: Combat, killing, looting
  • Result: Players solve mysteries through violence

Fix:

  • Add investigation mechanics
  • Reduce combat focus
  • Make violence counter-productive
  • OR accept this is combat game with mystery theme

Failure 5: Perspective Breaks

Symptom:

  • Horror game with comedy
  • Mystery with random solutions
  • Cozy with sudden threats

Example: Horror game with pratfalls

  • Perspective: Unknowable threat
  • Element: Alien fails at basic tasks (comedy)
  • Result: Fear becomes laughter

Fix:

  • Remove elements that break perspective
  • Maintain consistency of tone
  • Every element serves same feeling

Iteration and Refinement

Coherence rarely perfect on first attempt.

Step 1: Build Prototype

  • Implement core systems
  • Create sample content
  • Integrate context with mechanics

Step 2: Playtest for Contradictions

  • Do players behave as expected?
  • Do they report intended aesthetic?
  • Are there confusion points?
  • Any laughter in horror? Violence in cozy?

Step 3: Identify Incoherences

  • Which elements contradict?
  • Where do signals conflict?
  • What breaks perspective?

Step 4: Fix Systematically

If context contradicts mechanics:

  • Option A: Change context to match mechanics
  • Option B: Change mechanics to match context
  • Choose based on which is more fundamental to design

If resources contradict narrative:

  • Option A: Change resources (make scarce if desperate)
  • Option B: Change narrative (well-supplied if abundant)

If dynamics don't match intent:

  • Add mechanics that support intended behavior
  • Remove mechanics that create wrong behavior
  • Adjust incentives

Step 5: Retest

  • Validate fixes work
  • Check for new contradictions
  • Iterate until coherent

Final Integration Check

Before shipping, verify complete coherence:

The Five-Minute Test

Can you explain your game in five minutes and have it sound coherent?

"It's a [genre] game where players [expected dynamics] in a [context] setting. The aesthetic is [primary aesthetic], supported by [secondary aesthetics]. Players manage [key resources] while [main challenge]. Everything is designed to make players feel [perspective]."

Example - Horror: "It's a survival horror game where players hide from an alien threat in a spaceship setting. The aesthetic is Tension, supported by Challenge. Players manage limited health and ammunition while trying to reach escape pods. Everything is designed to make players feel hunted by an unknowable threat."

If this explanation feels contradictory → Not coherent If it flows naturally → Coherent

The Stranger Test

Can someone unfamiliar explain what your game is after 30 minutes of play?

Test questions to ask them:

  • "What kind of game is this?"
  • "What are you trying to do?"
  • "How does it feel?"
  • "What matters most?"

If answers match your intentions → Coherent If answers scattered or confused → Incoherent

The YouTube Test

Would streamers/viewers understand the game's identity from watching?

Watch someone play without explaining anything:

  • Do they get the intended feeling?
  • Do they exhibit intended behaviors?
  • Would viewers understand genre and aesthetic?

If yes → Coherent If confused → Incoherent

Summary: The Path to Coherence

Coherence is not accidental. It's systematic alignment.

The process:

  1. Start with aesthetic (Chapter 1, Section 2: The Twelve Aesthetics)
  2. Choose genre (Part 2)
  3. Define perspective (Part 4)
  4. Make content decisions (Part 5)
  5. Integrate resources (Chapter 2)
  6. Choose paradigm (Chapter 4)
  7. Set cognitive load (Chapter 3)
  8. Validate dynamics (Part 7)
  9. Check coherence (this part)
  10. Iterate until aligned

The goal: Every element serves the same aesthetic. No contradictions. Players intuitively understand what the game wants. Context transforms mechanics into meaning.

Weather Factory proves it's possible:

  • Same mechanics, perfect coherence in opposite aesthetics
  • Every element aligned
  • No contradictions
  • Players get exactly the intended experience

Your game can achieve this too.
