System Evolution: Synthesized Lessons

Summary of Four System Evolutions

D&D (1974-2024): The Pendulum Swing

  • OD&D → AD&D: Codification (wargame → RPG product)
  • AD&D → 3e: Unification (d20 system, everything uses same mechanic)
  • 3e → 4e: Balance obsession (powers for all, tactical focus) - FAILED
  • 4e → 5e: Synthesis (accessibility + tradition) - SUCCEEDED

Key insight: Can't just fix problems (4e was technically superior), must preserve feel

PbtA (2010-present): Purpose-Driven Design

  • One engine, many games: 2d6+stat is just resolution
  • Moves encode genre: Same mechanic, different fictional triggers
  • Resources match theme: Track what matters (Strings, Influence, Barter)
  • Fiction-first: Trigger from description, not skill list

Key insight: Simple core enables infinite variation through implementation

Civilization (1991-2016): Complexity Management

  • I-IV: Growing complexity (more systems = more depth BUT more tedium)
  • IV → V: Radical simplification (1UPT, removed systems) - DIVISIVE but successful
  • V → VI: Smart complexity (districts, eurekas) - BEST of both

Key insight: Complexity is good if managed well (spatial > numerical, guided > overwhelming)

XCOM (1994-2016): Modernization Through Simplification

  • Original: Deep but complex (TU system, 14 soldiers, overwhelming)
  • Enemy Unknown (2012): Radical simplification (2 actions, 4-6 soldiers, cover system)
  • XCOM 2 (2016): Smart complexity return (timed missions, concealment, more enemy types)

Key insight: Radical simplification can ADD depth if you preserve what matters

Complete Lesson Catalog

Category 1: Complexity and Accessibility

L1: Complexity is a budget, spend wisely

  • Source: All four systems
  • D&D 4e and Civ IV hit complexity ceiling
  • Adding systems has diminishing returns
  • Must justify each system's existence

L2: Simplification ≠ dumbing down

  • Source: XCOM, D&D 5e, Civ V
  • XCOM: TU system → 2 actions (simpler but deeper)
  • D&D 5e: Advantage/disadvantage (elegant replacement for stacking modifiers)
  • Can be more accessible AND deeper

L3: Simple core enables complex applications

  • Source: PbtA, D&D 5e
  • PbtA: 2d6+stat is trivial, moves are everything
  • D&D 5e: d20+mod is simple, application is complex
  • Complexity should be in use, not core mechanic

L4: Spatial/visual complexity > numerical complexity

  • Source: Civ VI, XCOM
  • Civ VI: District placement (spatial puzzle) vs Civ IV: Specialist optimization (spreadsheet)
  • XCOM: Cover positioning (visual/tactical) vs TU optimization (numerical)
  • Players prefer tangible to abstract

L5: Managed complexity works; unmanaged doesn't

  • Source: Civ VI, D&D 3.5e
  • Civ VI: Adjacency bonuses (complex but clear UI) vs Civ IV: (complex and opaque)
  • D&D 3.5e: Feat chains (complex and overwhelming)
  • Present complexity clearly or don't present it

L6: Guided complexity better than pure freedom

  • Source: Civ VI eurekas
  • Too much freedom = analysis paralysis
  • Guiding players toward good choices (while allowing alternatives) improves experience
  • Don't just give options, help players understand them

Category 2: Purpose and Identity

L7: Know what you're optimizing for

  • Source: All four systems
  • D&D 4e optimized for tactical balance, lost RPG feel
  • PbtA descendants optimize for different genres
  • Civ V optimized for accessibility, VI for smart complexity
  • XCOM always optimizes for tension/permadeath
  • Can't optimize for everything

L8: Purpose determines mechanics, not vice versa

  • Source: PbtA (strongest example)
  • Monsterhearts: Social warfare mechanics (Turn Someone On, Strings)
  • Dogs in the Vineyard: Escalation mechanics (moral choices)
  • Don't start with mechanics, start with "what experience?"

L9: Core identity must be preserved

  • Source: D&D, XCOM
  • D&D: Six attributes never changed (even when everything else did)
  • XCOM: Permadeath and tension always core
  • Sacred cows exist for a reason
  • Innovation within identity, not instead of it

L10: Balance alone is insufficient

  • Source: D&D 4e
  • 4e was extremely balanced (all classes worked similarly)
  • But felt wrong (lost D&D identity)
  • Players value feel over perfect balance
  • Asymmetry can be feature, not bug

L11: Can't just fix problems, must preserve magic

  • Source: D&D 4e
  • Fixed martial/caster gap, dead levels, skill bloat
  • Lost "essence" that made D&D feel like D&D
  • Technical superiority ≠ better game

Category 3: Resource Management

L12: Resources create meaningful choices through scarcity

  • Source: All four systems
  • HP (survival pressure), Spell slots (tactical decisions), Stress (escalation)
  • Abundant resources = no decisions
  • But too scarce = frustration
  • Find right scarcity level for purpose

L13: Track what matters for the experience

  • Source: PbtA, D&D evolution
  • Monsterhearts: Strings (social capital) not HP (combat resource)
  • D&D: HP always tracked (survival central), encumbrance often ignored (tedious)
  • Resources tell players what the game is about

L14: Different resources for different purposes

  • Source: D&D 5e, Blades in the Dark
  • HP = survival pressure (attrition)
  • Spell slots = tactical resource (per-encounter decisions)
  • Hit dice = risk/reward (spend now or save?)
  • Inspiration = roleplay reward (positive reinforcement)
  • Each serves distinct purpose

L15: Recovery cycles create pacing

  • Source: D&D, XCOM
  • No recovery = high stakes, slow pacing
  • Fast recovery = low stakes, fast pacing
  • D&D: Short rest (hour) vs Long rest (8 hours)
  • Defines gameplay rhythm

L16: In-game resources ≠ scenario constraints

  • Source: Our analysis (not explicitly in systems but important)
  • In-game: HP, ammo (managed BY system, player decisions)
  • Constraints: Player time, attention (constraints ON system, designer accommodates)
  • Don't conflate - require different tools

Category 4: Player Agency and Control

L17: Automation must be optional and transparent

  • Source: Civ, XCOM
  • Players refuse automation if it's not optimal
  • Need to feel in control
  • "Suggest" not "decide"
  • Show what automation will do

L18: Difficulty options respect player choice

  • Source: XCOM, D&D 5e
  • XCOM: Ironman (hardcore) vs saves (forgiving)
  • D&D 5e: Multiple difficulty suggestions
  • One difficulty doesn't fit all
  • Let players choose their challenge

L19: Transparency enables strategy

  • Source: XCOM EU/2, Civ VI
  • XCOM: Show hit chances, enemy abilities
  • Civ VI: Show adjacency bonuses clearly
  • Hidden info = frustration, not mystery
  • Players need info to make good decisions

L20: Players need to understand causality

  • Source: D&D 3.5e vs 4e, XCOM
  • D&D 3.5e: Complex but legible (if you know rules)
  • XCOM: Clear cause-effect (shoot in cover = miss more often)
  • Can't be black box
  • Must be able to learn from outcomes

Category 5: Pacing and Compulsion

L21: Short-term goals create "one more" compulsion

  • Source: Civ, D&D
  • Civ: "My tech finishes in 2 turns"
  • D&D: "One more encounter before rest"
  • Always something about to complete
  • Combines with long-term planning

L22: Positive feedback loops drive engagement

  • Source: Civ
  • More cities → more production → more cities (snowball)
  • But needs rubber-banding (catch-up mechanics)
  • Growth feels good
  • Compounding returns satisfying

L23: Failure should move story forward

  • Source: PbtA
  • 6- = GM makes move (complication, not nothing)
  • Never "try again, same outcome"
  • Maintains momentum
  • Failure is interesting

L24: Constraints force interesting decisions

  • Source: XCOM, PbtA
  • XCOM: Limited soldiers, actions, resources, time
  • Creates tension through scarcity
  • Every choice matters
  • Abundance = boring

L25: Structure can be genre

  • Source: PbtA (The Sprawl), Civ
  • Sprawl: Mission structure (Get Job → Legwork → Action) matches heist genre
  • AW: Sandbox structure matches post-apocalyptic chaos
  • Pacing is part of experience
  • Structure communicates genre

Category 6: Evolution and Iteration

L26: Evolution is reaction (each version responds to previous)

  • Source: D&D, Civ
  • D&D: 4e reacts to 3.5e complexity, 5e reacts to 4e alienation
  • Civ: V reacts to IV complexity, VI reacts to V simplicity
  • Pendulum swings
  • Overcorrection common

L27: Can simplify radically if core is strong

  • Source: XCOM, D&D 5e, Civ V
  • XCOM: Remove entire TU system, game still XCOM
  • Identity in purpose, not specific mechanics
  • Preserve what matters, change what doesn't

L28: Synthesis after thesis/antithesis

  • Source: D&D 5e, Civ VI
  • Thesis (3.5e complexity) → Antithesis (4e simplicity) → Synthesis (5e balance)
  • Civ: IV (complex) → V (simple) → VI (smart complexity)
  • Best solutions often middle path
  • But must try extremes to find it

L29: Same engine, different implementations

  • Source: PbtA
  • 2d6+stat is just resolution mechanic
  • Moves determine actual game
  • Can serve infinite purposes with same core
  • Implementation is everything

L30: Commercial success ≠ design quality (and vice versa)

  • Source: D&D 4e (good design, commercial failure), 5e (synthesis, huge success)
  • 4e was technically superior in many ways
  • But lost audience
  • Market matters
  • Audience must want what you're selling

Category 7: Specific Mechanics

L31: Advantage/disadvantage (simple elegance)

  • Source: D&D 5e
  • Roll 2d20, take higher/lower
  • Replaces stacking modifiers
  • Easy to adjudicate
  • 40 years to discover this simple solution

L32: Bounded accuracy (prevents inflation)

  • Source: D&D 5e
  • Modifiers capped, not infinite scaling
  • Low-level threats stay relevant
  • Math stays manageable
  • Prevents "I need +15 to hit AC 30" endgame

L33: Partial success as default

  • Source: PbtA
  • 7-9 most common result (58% on 2d6)
  • More interesting than binary success/fail
  • Creates complications
  • Maintains momentum

L34: Fiction-first triggers

  • Source: PbtA
  • Don't roll skill, describe action
  • "I threaten them" → triggers Go Aggro move
  • Mechanics emerge from fiction
  • Not reverse (mechanics → retrofit fiction)

L35: Moves encode genre/purpose

  • Source: PbtA
  • AW: Go Aggro (violence focus)
  • Monsterhearts: Turn Someone On (social focus)
  • Masks: Labels shift (identity focus)
  • Same 2d6, completely different games

L36: Stats are labels, not physics

  • Source: PbtA (especially Masks)
  • Cool/Hard/Hot (interaction types) not Str/Dex/Con (physical attributes)
  • Masks: Labels are "how others see you"
  • Names shape player thinking
  • Communication tool, not simulation

L37: Permadeath creates meaning

  • Source: XCOM, some TTRPGs
  • Soldiers you name become "yours"
  • Loss hurts
  • Makes you play carefully
  • Creates emergent stories
  • But must feel fair

L38: Cover system (clear tactical choices)

  • Source: XCOM EU
  • Visual indicators (shields)
  • Clear cause-effect (flanking, full vs half cover)
  • Simple to understand, hard to master
  • Made tactical positioning accessible

Category 8: UI and Presentation

L39: Interface is the game

  • Source: Civ evolution
  • Players spend hours looking at UI
  • Must be clear, beautiful, informative
  • Bad UI = bad game (even if mechanics good)
  • Presentation matters as much as systems

L40: Show what matters, hide what doesn't

  • Source: Civ VI, D&D 5e
  • Civ VI: Adjacency bonuses visible, complex math hidden
  • D&D 5e: Simple math (d20+mod), complex situations abstracted
  • Complexity can be back-end
  • Present clean choices to players

L41: Visual feedback teaches

  • Source: XCOM, Civ
  • XCOM: Hit chances, damage preview
  • Civ: "if you place here, you get +X"
  • Show consequences before committing
  • Players learn from clear feedback

Category 9: Content and Variety

L42: Procedural + authored balance

  • Source: XCOM, Civ
  • Procedural: Variety, prevents optimization
  • Authored: Quality, prevents broken situations
  • Need both
  • Pure procedural = shallow, pure authored = predictable

L43: Multiple victory paths create replayability

  • Source: Civ, D&D (different builds)
  • Civ: Military, Science, Culture, Diplomacy, Religion
  • Each requires different strategy
  • Creates meaningful choices
  • Prevents "one right way"

L44: Asymmetry creates variety

  • Source: Civ (civs have bonuses), PbtA (playbooks)
  • Not everyone has same options
  • Each playthrough different
  • Encourages exploration
  • But must be balanced

Category 10: LLM-Specific Insights

L45: LLMs can handle complexity humans can't

  • Source: Implied by all analyses
  • Complex state tracking (Civ-level)
  • Modifier calculation (D&D 3.5e-level)
  • NPC memory and consistency (impossible for human GM)
  • But must still present simply

L46: Natural language enables fiction-first

  • Source: PbtA analysis
  • Don't need skill list if LLM understands intent
  • "I try to intimidate them" → LLM identifies appropriate move
  • Removes need for codification
  • More flexible than menus

L47: LLMs can maintain principles invisibly

  • Source: PbtA GM Agenda/Principles
  • PbtA: GM follows Agenda (Make Apocalypse World seem real)
  • LLM can follow principles without player seeing framework
  • Creates consistent experience
  • Structured improvisation

L48: Adaptation vs agency tension

  • Source: All systems
  • LLM can adapt dynamically (difficulty, content)
  • But players need to feel in control
  • Must be transparent about adaptations
  • "Suggest" not "decide"

L49: Automation can remove tedium, not decisions

  • Source: Civ, XCOM
  • Automate bookkeeping (resource tracking)
  • Don't automate interesting choices (what to build)
  • Players need meaningful decisions
  • Tedium vs depth distinction critical

L50: Preserve rituals and agency

  • Source: All tabletop systems
  • Dice rolling is ritual (uncertainty, drama)
  • Character creation is identity expression
  • Don't automate everything just because you can
  • Some "inefficiency" is feature, not bug

Meta-Lessons (Patterns About Patterns)

ML1: Purpose must come first

  • Every successful design starts with "what experience?"
  • Mechanics serve purpose
  • Not "here's cool mechanic, what can we do with it?"
  • Space coffee problem

ML2: Evolution reveals truth

  • Early versions have assumptions baked in
  • Later versions question those assumptions
  • Failures teach as much as successes
  • 4e's failure taught as much as 5e's success

ML3: Simplicity is sophisticated

  • Advantage/disadvantage took 40 years
  • 2-action system was radical
  • Simple solutions are hard-won
  • Often emerge after complexity phase

ML4: Constraints breed creativity

  • PbtA's 2d6 limitation forced creative moves
  • XCOM's 2-action limit forced tactical thinking
  • Civ's district placement forced city planning
  • Limits create interesting decisions

ML5: Players know what they want (sometimes)

  • 4e backlash was real
  • But players also resist good changes initially (1UPT in Civ V)
  • Listen to players, but interpret carefully
  • "Players are great at identifying problems, terrible at proposing solutions"

ML6: Identity anchors innovation

  • D&D kept six attributes
  • XCOM kept permadeath
  • Can experiment wildly if core preserved
  • Sacred cows enable risk-taking elsewhere

ML7: Medium matters

  • TTRPGs: Human adjudication, social play, narrative focus
  • Video games: Computation, procedural content, immediate feedback
  • Same genre, different affordances
  • Design for medium, not against it

ML8: Commercial and artistic success can diverge

  • 4e was artistically bold, commercially failed
  • 5e less innovative, huge commercial success
  • Best games thread needle
  • But sometimes bold experiments necessary

ML9: There is no perfect system

  • Every design is tradeoffs
  • D&D 5e: Accessibility vs depth
  • XCOM: 2 actions vs flexibility
  • Civ VI: Complexity vs management
  • Know what you're trading off

ML10: Framework emerges from practice

  • These lessons came from analyzing real systems
  • Not from abstract theorizing
  • Grounded in what actually worked/failed
  • Practice → pattern → principle

Summary by System

D&D teaches: Identity preservation, complexity management, commercial realities PbtA teaches: Purpose-driven design, fiction-first mechanics, modular implementation Civ teaches: Complexity presentation, "one more turn" psychology, spatial thinking XCOM teaches: Radical simplification, risk/reward balance, modernization

Together they teach:

  • Purpose before mechanics
  • Simple core, complex application
  • Resources create decisions
  • Transparency enables strategy
  • Preserve identity while innovating
  • Respect player agency
  • Manage complexity through presentation
  • Know what you're optimizing for
  • Iteration reveals truth
  • There are no perfect solutions, only tradeoffs
