Part 11: Quick Reference - Aesthetic to Loop Checklist

This section provides quick-reference checklists for designing temporal architecture based on aesthetic goals. Use these as starting points, not rigid prescriptions. Your specific design context (target audience, platform, genre) will require adaptations.

How to Use These Checklists

For each aesthetic:

  1. Loop Requirements: What temporal structures does this aesthetic need?
  2. Investment Pattern: How should cognitive/time investment change across scales?
  3. Stopping Points: Where should players be able to stop satisfyingly?
  4. Return Mechanics: What brings players back (intrinsic only)?
  5. Red Flags: What temporal patterns break this aesthetic?

Process:

  1. Identify your primary aesthetic
  2. Review checklist for that aesthetic
  3. Check your loop architecture against requirements
  4. Validate stopping points match player capacity
  5. Ensure return mechanics are intrinsic

Discovery Aesthetic

Core Experience: Exploration, finding new things, understanding systems, revealing secrets

Loop Requirements

Micro loops (30-60 seconds):

  • ☐ Variety at each iteration (no two micro actions identical)
  • ☐ Information reveals (each action teaches something)
  • ☐ Clear feedback (immediately see what you learned)
  • ☐ No punishment for exploration (safe to try things)

Meso loops (5-30 minutes):

  • ☐ Exploration rewards (finding things feels good)
  • ☐ Gating that creates anticipation (visible locked content)
  • ☐ Recontextualization (new info changes earlier understanding)
  • ☐ Self-contained discoveries (can explore area and feel satisfied)

Macro loops (1-4 hours):

  • ☐ Secrets that gate progression (optional but rewarding)
  • ☐ "Aha!" moments (pieces click together)
  • ☐ Major capability unlocks (movement, access, understanding)
  • ☐ World/knowledge expansion (opens new areas or concepts)

Meta loops (10-40 hours):

  • ☐ Meta knowledge changes approach (knowing X makes Y different)
  • ☐ Optional completion (100% is for dedicated explorers)
  • ☐ Replayability through different paths (new discoveries each time)
  • ☐ No missable content (can't permanently lock out discoveries)

Investment Pattern

Trajectory: Flat Low to Learning Curve (depending on complexity)

  • Micro: Low (exploration should be simple)
  • Meso: Moderate (piecing together info requires thought)
  • Macro: Moderate (understanding systems/mysteries)
  • Meta: Low (optional depth for dedicated players)

Stopping Points

Frequency: Every 20-30 minutes (after discovery milestones)

  • ☐ Area complete (explored thoroughly)
  • ☐ Mystery solved (question answered)
  • ☐ Major discovery made (new capability/understanding)
  • ☐ Can resume easily (clear map/objective markers)

Return Mechanics (Intrinsic Only)

  • ✓ Curiosity about unlocked content (what's in that locked area?)
  • ✓ Unfinished mysteries (still have questions)
  • ✓ New capabilities to test (want to see what new tool does)
  • ✓ Environmental intrigue (world is interesting to explore)
  • ✗ No time-gated discoveries (no "explore today or miss forever")
  • ✗ No FOMO (all content always accessible)

Red Flags

  • ❌ Repetitive micro loops (kills Discovery—need variety)
  • ❌ Linear progression with no secrets (kills exploration drive)
  • ❌ Missable content without warning (creates anxiety, not curiosity)
  • ❌ Time pressure (Discovery needs time to explore)
  • ❌ Punishment for exploration (damages, costs resources)

Example Games

  • Outer Wilds: Pure Discovery, 22-minute loops force new approaches
  • Hollow Knight: Exploration rewards, equipment gates, backtracking reveals
  • Return of the Obra Dinn: Information discovery, deduction satisfaction

Challenge Aesthetic

Core Experience: Overcoming difficulty through skill development, earning victory through mastery

Loop Requirements

Micro loops (30-90 seconds):

  • ☐ Clear failure/success states (hit or miss, no ambiguity)
  • ☐ Consistent patterns (same input → same output, learnable)
  • ☐ Skill-based execution (timing, precision, pattern recognition)
  • ☐ Immediate feedback on mistakes (know what went wrong)

Meso loops (10-30 minutes):

  • ☐ Skill tests with escalation (gets harder as you progress)
  • ☐ Checkpoints as earned relief (reached through skill)
  • ☐ Resource attrition (consequences accumulate)
  • ☐ Clear victory conditions (know when section is complete)

Macro loops (30-120 minutes):

  • ☐ Difficulty progression (challenges build on earlier lessons)
  • ☐ Major skill gates (bosses, difficult sections)
  • ☐ Earned achievement (victory requires mastery)
  • ☐ Meaningful completion (major milestone reached)

Meta loops (20-100 hours):

  • ☐ Mastery recognition (achievements, rankings, community respect)
  • ☐ Optional harder challenges (New Game+, hard mode, speedruns)
  • ☐ Skill demonstration opportunities (ways to show expertise)
  • ☐ No required grinding (victory through skill, not time)

Investment Pattern

Trajectory: Learning Curve (high initial, low through mastery)

  • Micro: HIGH initially (learning controls, patterns) → LOW through mastery (muscle memory)
  • Meso: Moderate consistent (tactical decisions)
  • Macro: Moderate consistent (strategic planning)
  • Meta: Low (optional depth for mastery demonstration)

Critical: Micro investment MUST decrease with mastery (enables flow state)

Stopping Points

Frequency: Every 30-60 minutes (after earned achievements)

  • ☐ Checkpoint/bonfire reached (earned through skill)
  • ☐ Boss defeated (major victory)
  • ☐ Difficult section complete (sense of accomplishment)
  • ☐ Can resume at checkpoint (don't lose progress)

Return Mechanics (Intrinsic Only)

  • ✓ Boss revenge (want to defeat challenge that beat you)
  • ✓ Skill improvement satisfaction (getting better feels good)
  • ✓ Mastery drive (want to perfect execution)
  • ✓ Challenge variety (new bosses, new challenges)
  • ✗ No power treadmill (victory through skill, not grinding)
  • ✗ No required grinding (don't force time investment)

Red Flags

  • ❌ Random difficulty spikes (prevents pattern learning)
  • ❌ Victory through luck not skill (feels unearned)
  • ❌ Required grinding to progress (makes skill irrelevant)
  • ❌ Micro investment never decreases (prevents flow state)
  • ❌ Unclear failure causes (can't learn from mistakes)

Example Games

  • Dark Souls: Consistent patterns, earned checkpoints, skill-based victory
  • Sekiro: Precise timing, pattern learning, mastery satisfaction
  • Celeste: Pixel-perfect platforming with accessibility options

Expression Aesthetic

Core Experience: Creativity, self-expression, building/creating something unique

Loop Requirements

Micro loops (5-30 seconds):

  • ☐ Creation tools immediate (place block, draw line, write word)
  • ☐ Instant feedback (see result of action)
  • ☐ Easy undo (experimentation encouraged)
  • ☐ Low friction (creating is effortless)

Meso loops (10-60 minutes):

  • ☐ Build/test/iterate cycles (create → try → refine)
  • ☐ Playtest immediately (test creations instantly)
  • ☐ Refinement without punishment (iteration is core gameplay)
  • ☐ Self-contained projects (can complete in session)

Macro loops (1-4 hours):

  • ☐ Complete creations (finish level, song, building)
  • ☐ Sharing options (show others, get feedback)
  • ☐ Portfolio building (collection of works)
  • ☐ Clear completion (creation is done, can move to next)

Meta loops (ongoing):

  • ☐ Skill development visible (creations improve over time)
  • ☐ Community engagement (other creators, feedback, inspiration)
  • ☐ Style development (identity as creator)
  • ☐ Infinite replayability (always new things to create)

Investment Pattern

Trajectory: Learning Curve (mechanical skill) + Escalating Depth (creative skill)

  • Micro: Moderate initially (learning tools) → Low through practice (tools become automatic)
  • Meso: Moderate (creative decisions, iteration)
  • Macro: Variable (simple creations vs. ambitious projects)
  • Meta: High acceptable (creators enjoy deep engagement)

Stopping Points

Frequency: Self-determined (creator decides when done)

  • ☐ Creation complete (self-defined completion)
  • ☐ Can save work-in-progress (don't lose partial work)
  • ☐ Natural pauses (sections complete, good places to stop)
  • ☐ No pressure to finish (can abandon projects without loss)

Return Mechanics (Intrinsic Only)

  • ✓ New project ideas (inspired to create something)
  • ✓ Skill improvement (want to build better)
  • ✓ Community feedback (responses from others)
  • ✓ Creative satisfaction (expression itself is rewarding)
  • ✗ No time-limited creation (no "create now or lose access")
  • ✗ No creative FOMO (tools/options always available)

Red Flags

  • ❌ Complex creation tools (high friction kills flow)
  • ❌ Mandatory sharing (some creators are private)
  • ❌ Competitive ranking (contradicts personal expression)
  • ❌ Resource scarcity (limits creativity)
  • ❌ "Wrong" outcomes (all expression should be valid)

Example Games

  • Mario Maker: Build/test/share levels, community-driven
  • Minecraft Creative: Infinite resources, zero pressure, pure creation
  • Dreams: Complex creation tools, deep system, high investment acceptable

Mastery Aesthetic

Core Experience: Deep understanding, optimization, perfect execution, demonstrating expertise

Loop Requirements

Micro loops (1-3 seconds):

  • ☐ Deterministic (same input → same output always)
  • ☐ High skill ceiling (optimization possible)
  • ☐ Precision matters (small differences in execution matter)
  • ☐ Consistent rules (enables deep learning)

Meso loops (5-20 minutes):

  • ☐ Multiple solutions (optimization opportunities)
  • ☐ Complexity reveals over time (depth emerges)
  • ☐ Perfect execution possible (can achieve optimal play)
  • ☐ Strategy development (planning and execution both matter)

Macro loops (30-120 minutes):

  • ☐ Skill differentiation clear (experts vs. novices obvious)
  • ☐ Optimization opportunities (many ways to improve)
  • ☐ Performance measurement (rankings, times, scores)
  • ☐ Replayability through mastery (same content, better execution)

Meta loops (100-1000+ hours):

  • ☐ Infinite skill ceiling (can always improve)
  • ☐ Community mastery culture (experts share knowledge)
  • ☐ Competitive opportunities (tournaments, rankings)
  • ☐ No power creep (skill always matters, never obsoleted)

Investment Pattern

Trajectory: Escalating Depth (complexity reveals over time)

  • Micro: Moderate initially (learning execution) → Optimization focus (refinement never ends)
  • Meso: High (strategic depth emerges)
  • Macro: High (systems understanding deepens)
  • Meta: Very High acceptable (mastery players seek deep engagement)

Stopping Points

Frequency: Self-determined (mastery players choose engagement level)

  • ☐ Match/game complete (competitive endpoint)
  • ☐ Personal goal achieved (rank reached, time beaten)
  • ☐ Session fatigue (intense focus is exhausting)
  • ☐ Can always improve (no final endpoint, always more to master)

Return Mechanics (Intrinsic Only)

  • ✓ Skill improvement drive (want to get better)
  • ✓ Optimization satisfaction (finding better strategies)
  • ✓ Competitive challenge (facing skilled opponents)
  • ✓ Mastery demonstration (showing expertise)
  • ✗ No seasonal resets (don't invalidate mastery)
  • ✗ No power treadmill (skill always matters)

Red Flags

  • ❌ Randomness in outcomes (prevents optimization)
  • ❌ Hidden information (can't master what you can't understand)
  • ❌ Frequent rule changes (invalidates learned mastery)
  • ❌ Power creep (makes skill irrelevant)
  • ❌ No skill ceiling (everything becomes easy)

Example Games

  • Rocket League: Simple core, infinite skill ceiling, pure mastery
  • StarCraft II: Deterministic, deep complexity, competitive focus
  • Into the Breach: Perfect information, optimal solutions exist

Fellowship Aesthetic

Core Experience: Social connection, cooperation, shared achievement, playing together

Loop Requirements

Micro loops (10-60 seconds):

  • ☐ Low mechanical complexity (allows conversation during play)
  • ☐ Coordination opportunities (roles that complement)
  • ☐ Communication valuable (talking helps)
  • ☐ Shared feedback (team sees same information)

Meso loops (10-45 minutes):

  • ☐ Team objectives (must cooperate to succeed)
  • ☐ Coordination required (can't succeed alone)
  • ☐ Role differentiation (players have different capabilities)
  • ☐ Shared success/failure (team wins or loses together)

Macro loops (30-120 minutes):

  • ☐ Team achievement (victories are shared)
  • ☐ Comeback possible (losing team can recover)
  • ☐ Session appropriate (matches available playtime)
  • ☐ Social stopping points (team decides when to stop together)

Meta loops (ongoing):

  • ☐ Team identity formation (regular groups develop culture)
  • ☐ Social bonds deepen (friendship beyond game)
  • ☐ Flexible membership (can play with different groups)
  • ☐ No mandatory attendance (social, not obligation)

Investment Pattern

Trajectory: Flat Low to Moderate (social focus, not mechanical complexity)

  • Micro: Low (simple mechanics allow conversation)
  • Meso: Moderate (coordination requires thought)
  • Macro: Moderate (team strategy)
  • Meta: Low (social bonds are reward, not system mastery)

Stopping Points

Frequency: Social determination (team decides together)

  • ☐ Match/mission complete (natural endpoint)
  • ☐ Team consensus (everyone agrees to stop)
  • ☐ Session length appropriate (30-90 minutes typical)
  • ☐ No penalty for leaving (life happens, respect that)

Return Mechanics (Intrinsic Only)

  • ✓ Social connection (friends are playing)
  • ✓ Team success satisfaction (won together)
  • ✓ Positive culture (community is welcoming)
  • ✓ Shared goals (working toward something as team)
  • ✗ No social obligation (guilt-free absence)
  • ✗ No scheduled requirements (raids optional, not mandatory)

Red Flags

  • ❌ Toxic behavior enabled (ruins social experience)
  • ❌ Complex mechanics (prevents conversation)
  • ❌ Individual performance too important (creates blame)
  • ❌ Mandatory attendance (social becomes obligation)
  • ❌ Competitive pressure (stress replaces fun)

Example Games

  • Deep Rock Galactic: Cooperative PvE, positive culture, class synergy
  • Keep Talking and Nobody Explodes: Pure communication gameplay
  • Among Us: Social deduction, accessible mechanics, party focus

Submission Aesthetic

Core Experience: Relaxation, flow without pressure, "turning brain off," low-stakes enjoyment

Loop Requirements

Micro loops (continuous or very short):

  • ☐ Minimal cognitive load (simple or automatic)
  • ☐ Immediate gratification (constant positive feedback)
  • ☐ No failure states (or trivial failure)
  • ☐ Soothing repetition (meditative, not tedious)

Meso loops (1-5 minutes):

  • ☐ Brief strategic moments (quick choices, not stressful)
  • ☐ Return to flow quickly (pause then resume meditation)
  • ☐ Optional engagement (can ignore complexity)
  • ☐ Always progressing (no setbacks)

Macro loops (15-45 minutes):

  • ☐ Clear endpoints (prevents "one more turn" trap)
  • ☐ Hard time caps (respects player time)
  • ☐ Complete in short sessions (casual-friendly)
  • ☐ Always rewarding (never feel like wasted time)

Meta loops (20-60 hours):

  • ☐ Purely intrinsic motivation (curiosity only)
  • ☐ No FOMO mechanics (zero pressure to play)
  • ☐ No daily requirements (play when you want)
  • ☐ Optional completion (can engage as much/little as desired)

Investment Pattern

Trajectory: Flat Low (never increases)

  • Micro: Minimal (automatic or very simple)
  • Meso: Minimal (quick decisions)
  • Macro: Minimal (no complex strategy)
  • Meta: Minimal (pure curiosity)

Critical: Investment must NEVER increase (breaks Submission promise)

Stopping Points

Frequency: Very frequent (every 15-30 minutes)

  • ☐ Forced stops (time caps, daily limits)
  • ☐ Natural pauses (rounds complete, objectives reached)
  • ☐ Zero penalty for stopping (can quit anytime guilt-free)
  • ☐ Easy to resume (pick up instantly)

Return Mechanics (Intrinsic Only)

  • ✓ Habit formation (positive) (pleasant morning ritual)
  • ✓ Flow state satisfaction (felt good, want to feel that again)
  • ✓ Curiosity about unlocks (what's this new character like?)
  • ✓ Zero commitment (can play once or daily, no pressure)
  • ✗ No FOMO ever (most important for Submission)
  • ✗ No streaks that reset (creates anxiety)
  • ✗ No limited-time anything (pure optional engagement)

Red Flags

  • ❌ Time pressure (any urgency breaks Submission)
  • ❌ Complex decisions (cognitive load too high)
  • ❌ Failure punishment (creates stress)
  • ❌ FOMO mechanics (creates obligation)
  • ❌ Competitive elements (creates pressure)
  • ❌ Investment escalation (betrays aesthetic promise)

Example Games

  • Vampire Survivors: Automatic gameplay, meditative flow, hard time cap
  • Animal Crossing: Gentle rhythm, no pressure, real-time prevents binging
  • Wordle: 5 minutes daily, no FOMO (same puzzle for everyone), zero pressure

Narrative Aesthetic

Core Experience: Story engagement, character development, emotional moments, plot progression

Loop Requirements

Micro loops (30-120 seconds):

  • ☐ Don't interrupt narrative flow (simple mechanics)
  • ☐ Reinforce story themes (mechanics match narrative)
  • ☐ Environmental storytelling (world tells story during play)
  • ☐ Minimal friction (don't distract from story)

Meso loops (15-45 minutes):

  • ☐ Story beats (narrative moments at this scale)
  • ☐ Character development (relationships deepen)
  • ☐ Clear scene structure (beginning, middle, end)
  • ☐ Emotional pacing (quiet moments between intense scenes)

Macro loops (2-5 hours):

  • ☐ Chapter/act structure (major narrative arcs)
  • ☐ Dramatic escalation (stakes increase)
  • ☐ Climax moments (emotional peaks)
  • ☐ Resolution phases (breathing room after climax)

Meta loops (10-40 hours):

  • ☐ Overall story arc (complete narrative)
  • ☐ Character journeys (growth and change)
  • ☐ Thematic exploration (story says something)
  • ☐ Satisfying ending (narrative closure)

Investment Pattern

Trajectory: Learning Curve (mechanics) + Growing Payoff (narrative)

  • Micro: Low (mechanics shouldn't distract)
  • Meso: Moderate (engaged in story)
  • Macro: High (emotionally invested)
  • Meta: High (deeply invested in characters/outcome)

Stopping Points

Frequency: Story-appropriate (after narrative beats)

  • ☐ Chapter/scene complete (narrative closure)
  • ☐ Cliffhangers intentional (drives return)
  • ☐ Emotional moments resolve (processing time)
  • ☐ Can save anywhere (respect player schedules)

Return Mechanics (Intrinsic Only)

  • ✓ Story intrigue (what happens next?)
  • ✓ Character investment (care about people)
  • ✓ Emotional engagement (story affects you)
  • ✓ Cliffhanger pull (need resolution)
  • ✗ No time-gated story (story available when player is ready)
  • ✗ No episodic releases (unless that's the format)

Red Flags

  • ❌ Complex mechanics distract from story
  • ❌ Gameplay contradicts narrative
  • ❌ Difficulty gates story access (skill shouldn't block narrative)
  • ❌ Long gameplay sections without story
  • ❌ Mechanics don't reinforce themes

Example Games

  • The Last of Us: Gameplay serves narrative, strong characters, emotional impact
  • Disco Elysium: Mechanics ARE narrative, writing-focused, choices matter
  • What Remains of Edith Finch: Mechanics change per vignette, narrative-driven

Tension Aesthetic

Core Experience: Adrenaline, time pressure, high-stakes decisions, nail-biting moments

Loop Requirements

Micro loops (10-60 seconds):

  • ☐ Constant threat (danger always present)
  • ☐ Mistakes costly (consequences immediate)
  • ☐ Limited margin for error (must execute carefully)
  • ☐ Paranoia encouraged (always checking, listening)

Meso loops (10-30 minutes):

  • ☐ Building pressure (stakes escalate)
  • ☐ Resource scarcity (limited tools)
  • ☐ Brief relief moments (safe rooms, checkpoints)
  • ☐ Risk/reward decisions (take chance or play safe?)

Macro loops (30-120 minutes):

  • ☐ Escalating danger (threats increase)
  • ☐ Attrition pacing (resources depleting)
  • ☐ Climax moments (peak intensity)
  • ☐ Cathartic relief (pressure releases)

Meta loops (10-30 hours):

  • ☐ Sustained tension possible (game doesn't become easy)
  • ☐ Mastery helps but doesn't remove threat
  • ☐ Optional difficulty for tension seekers
  • ☐ Complete story arc (narrative resolution)

Investment Pattern

Trajectory: Learning Curve + Sustained High (tension requires focus)

  • Micro: High (constant vigilance)
  • Meso: High (sustained focus)
  • Macro: High (no relaxation until climax)
  • Meta: Moderate (can take breaks between sessions)

Stopping Points

Frequency: After relief moments (30-60 minutes)

  • ☐ Safe room reached (pressure released)
  • ☐ Major threat defeated (breathing room)
  • ☐ Checkpoint/save point (progress secure)
  • ☐ Emotional exhaustion (tension is draining)

Return Mechanics (Intrinsic Only)

  • ✓ Adrenaline seeking (enjoyed the fear)
  • ✓ Overcoming fear satisfaction (conquered terror)
  • ✓ Story resolution drive (need to see how it ends)
  • ✓ Mastery of systems (learning enemy patterns)
  • ✗ No time pressure to return (Tension in-game, not meta)

Red Flags

  • ❌ Constant tension without relief (exhausting)
  • ❌ Jump scares instead of tension (cheap fear)
  • ❌ No resource management (removes scarcity tension)
  • ❌ Too much safety (kills tension)
  • ❌ Predictable threats (tension requires uncertainty)

Example Games

  • Alien: Isolation: Constant threat, resource scarcity, sustained tension
  • FTL: Time pressure, tough decisions, high stakes
  • Darkest Dungeon: Attrition, stress management, mounting pressure

Progression Aesthetic

Core Experience: Growth, becoming more powerful, visible advancement, unlocking capabilities

Loop Requirements

Micro loops (30-90 seconds):

  • ☐ Feel slightly better over time (gradual power increase)
  • ☐ New capabilities emerge (unlock abilities)
  • ☐ Visible growth (numbers going up, effects stronger)
  • ☐ Early game feels weak (creates contrast)

Meso loops (10-30 minutes):

  • ☐ Regular power spikes (upgrades, level-ups)
  • ☐ New options unlock (equipment, skills, areas)
  • ☐ Visible progress bars (XP, completion %)
  • ☐ Satisfying milestones (new tier reached)

Macro loops (1-3 hours):

  • ☐ Major power increases (transformative upgrades)
  • ☐ New gameplay opens up (abilities change approach)
  • ☐ Clear progression path (can see what's next)
  • ☐ Boss victories enable advancement (gates overcome)

Meta loops (20-100 hours):

  • ☐ Multiple progression systems (stats, equipment, skills, unlocks)
  • ☐ Always advancing something (constant progression)
  • ☐ Late game feels powerful (payoff for investment)
  • ☐ Optional post-game content (hard mode, New Game+)

Investment Pattern

Trajectory: Moderate Escalating (deepening engagement acceptable)

  • Micro: Moderate (execution gets easier as you get stronger)
  • Meso: Moderate (build decisions)
  • Macro: Moderate to High (long-term planning)
  • Meta: High acceptable (progression players enjoy deep engagement)

Stopping Points

Frequency: After progression milestones (30-60 minutes)

  • ☐ Level up achieved (visible milestone)
  • ☐ Major upgrade acquired (new capability)
  • ☐ Area/tier complete (progression gate passed)
  • ☐ Can see next goal (know what you're working toward)

Return Mechanics (Intrinsic Only)

  • ✓ Progression continuation (working toward next milestone)
  • ✓ Build experimentation (trying new combinations)
  • ✓ Power fantasy (feeling stronger)
  • ✓ Clear goals (can see what's next)
  • ✗ No power resets (don't invalidate progress)
  • ✗ No time-limited progression (can advance at own pace)

Red Flags

  • ❌ Progression trivializes content (no longer challenging)
  • ❌ Too many progression systems (overwhelming)
  • ❌ No progression for long periods (feels stagnant)
  • ❌ Progression locked behind RNG (frustrating)
  • ❌ Power resets (invalidates investment)

Example Games

  • Hades: Multiple progression systems, constant advancement, permanent upgrades
  • Monster Hunter: Equipment-based progression, visible power increases
  • RPGs generally: Leveling, skill trees, equipment upgrades

Multi-Aesthetic Games

Many games combine multiple aesthetics. When designing for multiple aesthetics:

Prioritization

Identify primary aesthetic:

  • This aesthetic wins conflicts
  • Determines core loop structure
  • Shapes overall feel

Support secondary aesthetics:

  • Must not contradict primary
  • Add depth without confusion
  • Optional engagement possible

Example: Hades

Primary: Challenge (core combat)

  • Micro loops: Skill-based combat
  • Meso loops: Room difficulty progression
  • Investment: Learning Curve at micro scale

Secondary: Progression (meta unlocks)

  • Permanent upgrades help but don't trivialize
  • Always advancing something
  • Rewards investment over time

Secondary: Narrative (relationship progression)

  • Story advances through death (aligns with Challenge failure)
  • Optional engagement (can skip dialogue)
  • Adds depth without complexity

Why it works: All aesthetics align. Death serves all three (Challenge test, Progression currency, Narrative advancement). No contradictions.

Red Flags for Multi-Aesthetic

  • ❌ Aesthetics contradict (Challenge needs failure, Submission needs success)
  • ❌ Too many aesthetics (confusing, unfocused)
  • ❌ Secondary aesthetic becomes mandatory (breaks optional nature)
  • ❌ Investment patterns conflict (Submission needs low, Mastery needs high)

Platform and Context Considerations

Mobile

Constraints:

  • Short sessions (5-30 minutes)
  • Interruptions frequent (pause anytime)
  • Touch controls (simple mechanics)
  • Battery/data conscious

Loop implications:

  • Micro loops: Very simple
  • Meso loops: 5-10 minutes max
  • Macro loops: 15-30 minutes
  • Meta loops: Optional, not required

Console/PC

Constraints:

  • Longer sessions possible (30-180 minutes)
  • Fewer interruptions (dedicated time)
  • Complex controls available (if desired)
  • TV/monitor setup (different posture)

Loop implications:

  • Micro loops: Can be more complex
  • Meso loops: 10-30 minutes acceptable
  • Macro loops: 1-3 hours possible
  • Meta loops: Deep engagement supported

Browser

Constraints:

  • Tab-switching common (interruptions)
  • Work/school context (short sessions)
  • Quick loading essential
  • No installation commitment

Loop implications:

  • Micro loops: Instant engagement
  • Meso loops: 3-10 minutes
  • Macro loops: 10-30 minutes max
  • Meta loops: Light (account-based okay)

Using This Reference

Design Process:

  1. Identify primary aesthetic (this section)
  2. Review loop requirements checklist
  3. Design loop architecture (Part 2-4)
  4. Validate against investment pattern (Part 5)
  5. Design stopping points (Part 6)
  6. Operationalize to components (Part 10)

Validation Process:

  1. Check all boxes in requirements section
  2. Verify investment pattern matches
  3. Test stopping points with target audience
  4. Ensure return mechanics are intrinsic
  5. Watch for red flags in playtesting

Remember: These are starting points, not rigid rules. Your specific context (audience, platform, genre) may require adaptations. The goal is coherent configuration serving your aesthetic goals while respecting your players' capacity.
