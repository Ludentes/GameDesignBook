Supplement 3: Temporal Architecture Case Studies

Purpose: Demonstrate complete temporal architecture design process through three invented game scenarios

This supplement shows:

  • Complete derivation from aesthetic → temporal architecture → validation
  • Different aesthetic/audience/medium combinations
  • How to handle real design constraints
  • Integration with resources, cognitive load, and balance considerations

How to Use These Case Studies

Each case study follows this structure:

  1. Design Brief - The initial request/constraints
  2. Aesthetic Analysis - What experience we're creating
  3. Temporal Requirements Derivation - From aesthetic to temporal properties
  4. Loop Architecture Design - Specific duration decisions
  5. Pacing Pattern Selection - Which of five patterns
  6. Stopping Point Design - Respecting player time
  7. Integration Analysis - Resources, cognitive load, balance
  8. Validation Plan - How to test and iterate
  9. Expected Measurables - Predicted outcomes

These are invented games designed to demonstrate temporal architecture principles across different contexts.

Case Study 1: "Neon Vanguard" - Mobile Action Game

Design Brief

Client Request: "We want a mobile action game for the casual-to-midcore audience. Think Vampire Survivors meets cyberpunk aesthetic. Players should be able to play during commute (10-20 minute sessions) but have reason to return daily. Monetization through cosmetics and optional battle pass."

Fixed Constraints:

  • Platform: Mobile (iOS/Android)
  • Sessions: 10-30 minutes maximum
  • Audience: Casual to midcore
  • Must support interruptions (phone calls, transit stops)
  • Free-to-play model

Negotiable:

  • Exact combat style
  • Progression depth
  • Art style specifics (cyberpunk direction given)

Design Freedom:

  • System design
  • Temporal architecture
  • Progression systems

Aesthetic Analysis

Primary Aesthetic: Challenge (light)

  • Overcoming waves of enemies through positioning and power selection
  • Pattern recognition (enemy types, spawn patterns)
  • Skill development possible but not required for casual

Secondary Aesthetic: Progression

  • Unlocking new abilities and characters
  • Visible power growth
  • Collection satisfaction

Tertiary Aesthetic: Submission

  • Can play while partially distracted
  • Combat mostly automatic (like Vampire Survivors)
  • Flow state achievable

Audience Capacity:

  • Casual majority: 10-20 minute sessions, low cognitive load tolerance, interruption-prone
  • Midcore subset: 20-30 minute sessions, moderate complexity acceptable, want depth

Target: Design primarily for casual, with optional midcore depth (don't punish casual for not engaging with depth)

Temporal Requirements Derivation

From Challenge aesthetic:

  • Fast iteration (but less critical than hardcore Challenge)
  • Clear feedback on success/failure
  • Learnable enemy patterns
  • Mastery demonstration through performance

From Progression aesthetic:

  • Frequent rewards (maintain engagement)
  • Visible advancement (unlocks, levels)
  • Meta-progression across sessions
  • Clear milestones

From Submission aesthetic:

  • Low execution pressure
  • Automatic elements reduce cognitive load
  • Flow state possible
  • No timing-critical actions

From Casual Audience:

  • Sessions must complete in 10-20 minutes
  • Can stop at ANY point without loss
  • Resume instantly when returning
  • Minimal between-session memory requirements

From Mobile Platform:

  • Portrait orientation (one-handed possible)
  • Touch controls (no precision required)
  • Pause-anywhere mandatory
  • Battery conscious (not graphically intensive combat)

Synthesized Temporal Requirements:

  1. Fixed-duration runs (predictability)
  2. Very simple micro-loops (one-hand touch)
  3. Frequent meso payoffs (rewards every 1-2 minutes)
  4. Can pause/resume instantly
  5. Zero time pressure
  6. Meta-progression visible and motivating

Loop Architecture Design

Micro Loop: 3-8 seconds

Design Decision:

  • Player action: Movement only (drag finger)
  • Automatic: Weapons fire, collection, targeting
  • Cycle: Move → Weapons fire → Damage feedback → Enemies defeated → XP collected → Move
  • Duration: 3-8 seconds depending on enemy density

Rationale:

  • One-handed touch control (drag to move)
  • No precision timing (forgiving)
  • Automatic combat (low cognitive load)
  • Enables play while distracted (commute)
  • Can execute while in flow state

Meso Loop: 60-90 seconds (level-up)

Design Decision:

  • Every level-up = meso loop completion
  • XP bar fills gradually
  • Level-up offers 3 ability choices
  • Choice pauses action (thinking time)
  • Approximately every 60-90 seconds

Rationale:

  • Frequent positive feedback (every 1-2 minutes)
  • Choice adds light strategy (midcore engagement)
  • Pause for choice = interruption-friendly
  • Visible progress toward next level-up

Macro Loop: 10-15 minutes (run)

Design Decision:

  • Fixed run duration: 12 minutes
  • Timer visible (countdown)
  • Win condition: Survive 12 minutes
  • Difficulty escalates throughout
  • End = guaranteed stopping point

Rationale:

  • Fits commute length (bus/train ride)
  • Predictable duration (can plan real life)
  • Always ends (respects player time)
  • Escalating difficulty creates arc
  • Fixed duration = perfect for casual

Meta Loop: 2-5 hours per unlock milestone

Design Decision:

  • Permanent upgrades purchased with run currency
  • New characters unlock every 4-6 runs (casual play)
  • New abilities unlock regularly
  • Battle pass (optional) provides cosmetics
  • No daily requirements (intrinsic motivation only)

Rationale:

  • Provides return motivation (unlock progression)
  • Pace reasonable for casual (not grind-heavy)
  • Optional battle pass doesn't gate power
  • No FOMO mechanics (respect player autonomy)

Failure Recovery: 5 seconds

Design Decision:

  • Death ends run immediately
  • Run results shown (currency earned, time survived)
  • "Try Again" button prominent
  • No menus required to restart

Rationale:

  • Fast restart maintains engagement
  • Clear feedback on performance
  • No punishment beyond lost run
  • Respects "quick session" goal

Pacing Pattern Selection

Primary: Escalating Intensity

Implementation:

  • Run starts calm (low enemy density)
  • Every 2 minutes: Difficulty spike (more enemies, faster, tougher)
  • Minute 8-12: High intensity (screen nearly full)
  • Climax at 12 minutes
  • End or death = full reset

Why this pattern:

  • Matches Challenge aesthetic (escalating difficulty)
  • Creates dramatic arc (calm → intense → climax)
  • Predictable structure (player knows what's coming)
  • Escalation manageable in 12 minutes (won't exhaust)

Secondary: Episodic (between runs)

  • Each run = complete episode
  • Clear beginning (start), middle (escalation), end (victory/defeat)
  • Next run = fresh start
  • Progression persists but arc resets

Stopping Point Design

Primary Stopping Points: End of every run (12 minutes or death)

Six elements check:

  1. Clear completion signal: ✓

    • "Victory!" or "Defeated" screen
    • Fanfare, visual celebration
    • Run statistics displayed
    • Currency earned shown

  2. Progress persistence: ✓

    • All currency saved automatically
    • Unlocks permanent
    • No progress loss ever
    • Cloud save for device switching

  3. State simplification: ✓

    • Run complete = no ongoing state
    • Start fresh next run
    • Only meta-progression tracked
    • No mid-run state to remember

  4. Emotional resolution: ✓

    • Victory or death = arc complete
    • No cliffhangers
    • Satisfying regardless of outcome
    • Next run = new story

  5. Re-entry clarity: ✓

    • Start new run = clean slate
    • UI shows available upgrades
    • New abilities highlighted
    • No confusion possible

  6. Time visibility: ✓

    • Timer shows 12:00 → 00:00 throughout run
    • Always know time remaining
    • Can decide "one more run?" with full information
    • No hidden duration

Result: Excellent stopping points every 12 minutes maximum

Secondary Stopping Point: Between runs (instant)

  • Can exit app mid-run
  • Run continues from pause on return
  • Can abandon run without penalty
  • Ultra-casual friendly

Integration Analysis

Resources (Chapter 3):

In-run resources:

  • Health: Regenerates slowly, can be upgraded
  • XP: Accumulates constantly, never lost
  • No resource scarcity pressure (Submission aesthetic)

Meta resources:

  • Currency: Earned per run, used for unlocks
  • No decay or loss mechanisms
  • Abundant (casual players can progress)
  • Accelerators available (optional monetization)

Recovery rates:

  • Health: 10 HP per second (frequent small hits okay)
  • XP: Continuous stream (level-up every 60-90s)
  • Supports Submission aesthetic (no scarcity stress)

Cognitive Load (Chapter 4):

Tracking load: Very Low

  • Health bar (single number, always visible)
  • XP bar (single progress bar, always visible)
  • Current level (single number)
  • Timer (12:00 → 00:00)
  • Total: 4 elements, all visual, no mental math

Decision load: Low

  • Micro: Movement only (where to position?)
  • Meso: 1 choice per 60-90s (3 options, clear descriptions)
  • Macro: Strategic (which upgrades to buy?)
  • Meta: Collection (which character to unlock next?)

Execution load: Very Low

  • Single input: Touch-drag movement
  • No timing windows
  • No precision requirements
  • Automatic combat entirely

Total cumulative load: Very Low

  • Appropriate for casual audience ✓
  • Can play while distracted ✓
  • Midcore can optimize (build choices) but not required ✓

Balance (Chapter 6 - Forward Looking):

Paradigm: Success-Based

  • Casual players should reach 12 minutes often
  • Challenge from optimizing time-to-12-minutes
  • Death possible but not punishing
  • Skill ceiling through ability synergies (midcore engagement)

Difficulty curve:

  • Minutes 0-4: Easy (learn controls, warm up)
  • Minutes 4-8: Medium (engaging but manageable)
  • Minutes 8-12: Hard (intense but achievable)
  • Victory rate target: 60-80% for casual, 90%+ for engaged

Progression impact:

  • Permanent upgrades ease difficulty gradually
  • New players struggle initially (die at 6-8 minutes)
  • Experienced players reach 12 minutes consistently
  • Mastery = speed, efficiency, style (leaderboards optional)

Validation Plan

Phase 1: Micro Loop Testing (Week 1-2)

Test focus:

  • Touch controls feel good?
  • One-handed play viable?
  • Can execute while distracted?
  • Automatic combat satisfying?

Success criteria:

  • Micro cycle 3-8 seconds ✓
  • Players report smooth execution ✓
  • No frustration with controls ✓
  • Can play one-handed ✓

Phase 2: Meso Loop Testing (Week 3-4)

Test focus:

  • Level-up frequency appropriate?
  • Choices interesting but not overwhelming?
  • Progression feels rewarding?

Success criteria:

  • Level-up every 60-90 seconds ✓
  • 80%+ find choices clear ✓
  • Satisfaction with progression ✓

Phase 3: Macro Loop Testing (Week 5-6)

Test focus:

  • 12-minute duration appropriate?
  • Escalation curve feels good?
  • End provides closure?
  • Stopping points satisfactory?

Success criteria:

  • Session duration 10-15 minutes (including menu time) ✓
  • Escalation feels dramatic but manageable ✓
  • 80%+ report satisfying endings ✓
  • Can comfortably stop after any run ✓

Phase 4: Meta Loop Testing (Week 7-10)

Test focus:

  • Return rate high?
  • Progression pace appropriate?
  • No FOMO or obligation feelings?
  • Sustainable engagement?

Success criteria:

  • 70%+ return after 1 week ✓
  • Progression feels achievable (not grindy) ✓
  • <10% report obligation feelings ✓
  • Retention at 2 weeks ≥ 50% ✓

Expected Measurables

Duration Measurables:

  • Micro loop cycle time: 3-8 seconds
  • Meso loop (level-up): 60-90 seconds
  • Macro loop (run): 12 minutes fixed
  • Failure recovery: 5 seconds

Frequency Measurables:

  • Level-ups per run: 8-10
  • Runs per session: 1-2 (casual), 2-4 (midcore)
  • Sessions per week: 3-5 (casual), 5-7 (midcore)

Pacing Measurables:

  • Intensity variance: Medium (escalating pattern)
  • Player-reported intensity: Increases linearly
  • Relief moments: Between runs only

Stopping Point Measurables:

  • Stopping point frequency: Every 12 minutes
  • Stopping point quality: 6/6 elements present
  • Exit utilization: 90%+ at run end
  • Mid-run exits: Only forced (interruptions)

Engagement Measurables:

  • Retention 1 week: 70%+
  • Retention 2 weeks: 50%+
  • Retention 4 weeks: 35%+
  • Return motivation: 80%+ intrinsic

Design Outcomes Summary

"Neon Vanguard" temporal architecture successfully:

✓ Serves Challenge aesthetic (light difficulty, pattern learning) ✓ Serves Progression aesthetic (unlocks, visible growth) ✓ Serves Submission aesthetic (automatic combat, flow state) ✓ Matches casual audience capacity (10-15 minute sessions, low cognitive load) ✓ Respects mobile platform constraints (one-hand, pause-anywhere) ✓ Provides excellent stopping points (every 12 minutes, 6/6 elements) ✓ Enables optional midcore engagement (build optimization) ✓ Avoids exploitative mechanics (no FOMO, no daily requirements)

This demonstrates Challenge + Progression + Submission for casual mobile audience with ethical temporal architecture.

Case Study 2: "Midnight Archives" - PC Mystery Investigation Game

Design Brief

Client Request: "We want a detective mystery game for players who loved Return of the Obra Dinn and Outer Wilds. Investigation-focused, no combat. Midcore audience who can invest 2-3 hour sessions. Setting is a supernatural archive/library. Players piece together what happened from records, artifacts, and echoes."

Fixed Constraints:

  • Platform: PC (Steam)
  • Audience: Midcore (2-3 hour sessions typical)
  • Genre: Investigation/mystery
  • No combat or time pressure
  • Single-player

Negotiable:

  • Exact investigation mechanics
  • Number of cases/mysteries
  • Art style (as long as atmospheric)

Design Freedom:

  • Temporal structure
  • Investigation systems
  • Progression between cases

Aesthetic Analysis

Primary Aesthetic: Discovery

  • Uncovering what happened through investigation
  • Connecting clues across multiple observations
  • "Aha!" moments when patterns recognized
  • Mystery-solving satisfaction

Secondary Aesthetic: Narrative

  • Story revealed through investigation
  • Character histories emerge from clues
  • Emotional beats when truths uncovered
  • Tragedy and mystery atmosphere

Tertiary Aesthetic: Challenge (light)

  • Deduction difficulty (not execution difficulty)
  • Some mysteries harder than others
  • Satisfaction from solving tough cases
  • Optional hint system available

Audience Capacity:

  • Midcore: 2-3 hour sessions, can return days later
  • High cognitive tolerance (tracking many clues)
  • Enjoys complex systems
  • Values depth over accessibility

Target: Design for engaged, thoughtful play with complex investigations

Temporal Requirements Derivation

From Discovery aesthetic:

  • No time pressure (observation requires time)
  • Player controls pacing entirely
  • Can revisit any location/clue anytime
  • Memory offloaded to systems (notebook, connections)
  • Pattern recognition space

From Narrative aesthetic:

  • Story unfolds gradually
  • Emotional beats properly paced
  • Revelations feel earned
  • Can reflect on story implications

From Challenge (light) aesthetic:

  • Some deductions harder than others
  • Multiple difficulty levels (easy observations, hard connections)
  • Hint system optional
  • No failure state (can't "lose")

From Midcore Audience:

  • Sessions 2-3 hours comfortable
  • Can maintain complex mental models
  • Willing to take notes (but systems should help)
  • Returns days later (needs recap support)

From PC Platform:

  • Keyboard/mouse (precise interaction)
  • Stationary play (deep focus possible)
  • Save system robust
  • Alt-tab friendly (players may research, take notes externally)

Synthesized Temporal Requirements:

  1. Micro loops: Variable duration (observation-driven)
  2. Meso loops: 30-50 minutes (per investigation area)
  3. Macro loops: 2-4 hours (per major case)
  4. No time pressure anywhere
  5. Complete freedom of exploration
  6. Strong systems for memory offloading
  7. Stopping points after revelations
  8. Return clarity essential

Loop Architecture Design

Micro Loop: 1-4 minutes (observation/examination)

Design Decision:

  • Examine artifact: 30-90 seconds (read description, look for details)
  • Read document: 1-3 minutes (journal entries, records)
  • Listen to echo: 1-2 minutes (ghostly conversation replay)
  • Take notes: 30 seconds (system prompts for notation)

Rationale:

  • Variable duration = player controls depth
  • No interruption during observation (can read completely)
  • Note-taking integrated (reduces cognitive load)
  • Can revisit anytime (no FOMO on clues)

Meso Loop: 30-50 minutes (area investigation)

Design Decision:

  • Each archive section contains 10-15 clue points
  • Area investigation = find all clues + form hypotheses
  • Hypothesis board opens when sufficient clues gathered
  • Revelation when correct connections made

Rationale:

  • Substantial investigation time (midcore capacity)
  • Multiple clues build toward understanding
  • Hypothesis system guides without solving
  • Revelation provides satisfying closure

Macro Loop: 2.5-4 hours (complete case)

Design Decision:

  • Each case spans 4-6 archive sections
  • Case structure: Introduction → Investigation → Connections → Revelation
  • Major revelation = emotional story payoff
  • Clear narrative closure (but seeds next mystery)

Rationale:

  • Fits midcore session capacity (single case per session or two)
  • Complete story arc per case
  • Stopping point after revelation (natural exit)
  • Seeds next case (return motivation)

Meta Loop: 20-30 hours (all cases + final mystery)

Design Decision:

  • 8-10 major cases
  • Each reveals piece of larger mystery
  • Final case synthesizes all previous
  • Meta-narrative about archive itself
  • Completionist content (optional deep dives)

Rationale:

  • Sustained engagement for midcore
  • Each case stands alone but connects
  • Final synthesis = major payoff
  • Optional content for Discovery enthusiasts

Failure Recovery: N/A

Design Decision:

  • No failure state exists
  • Wrong deductions noted but not punished
  • Can revise hypotheses freely
  • Hint system available (doesn't break immersion)

Rationale:

  • Discovery aesthetic = exploration, not challenge
  • Players learn through experimentation
  • No frustration from "wrong answers"
  • Supports midcore who want to figure it out

Pacing Pattern Selection

Primary: Emergent (player-controlled)

Implementation:

  • Player decides investigation order
  • Can deep-dive or skim as desired
  • Some areas intense (active deduction)
  • Some areas calm (reading, absorbing)
  • No forced pacing

Why this pattern:

  • Discovery aesthetic requires freedom
  • Some players want to rush revelations
  • Some want to savor every detail
  • Emergent pacing accommodates both

Secondary: Episodic (between cases)

  • Each case = complete episode
  • Clear beginning (case opened)
  • Middle (investigation)
  • End (revelation)
  • Can pause between cases

Stopping Point Design

Primary Stopping Points: After case revelations (every 2.5-4 hours)

Six elements check:

  1. Clear completion signal: ✓

    • "Case Solved" screen
    • Revelation cutscene/sequence
    • Archive updates with new information
    • Achievement/completion marker

  2. Progress persistence: ✓

    • All clues and hypotheses saved
    • Notebook fully persistent
    • Any case can be revisited
    • Cloud save for multiple PCs

  3. State simplification: ✓

    • Case closed = investigation complete
    • New case = fresh investigation
    • No ongoing complex state
    • Archive overview shows all cases

  4. Emotional resolution: ✓

    • Revelation provides story closure
    • Character fates revealed
    • Emotional beats complete
    • Seeds next mystery (anticipation, not cliffhanger)

  5. Re-entry clarity: ✓

    • Case log shows status
    • Notebook accessible (all previous notes)
    • Can review any case anytime
    • Current objectives highlighted

  6. Time visibility: ✓

    • Case structure visible (4-6 sections)
    • Section progress shown
    • Estimated remaining: Implicit from sections left
    • Can gauge "one more section or stop?"

Result: Excellent stopping points every 2.5-4 hours

Secondary Stopping Points: After area investigations (every 30-50 minutes)

Partial elements:

  • Completion signal: ✓ (area revelation)
  • Progress persistence: ✓ (all saved)
  • State simplification: Partial (case ongoing)
  • Emotional resolution: Partial (minor beats)
  • Re-entry clarity: ✓ (notebook)
  • Time visibility: Partial (sections visible)

Result: Acceptable mid-case stopping points for shorter sessions

Integration Analysis

Resources (Chapter 3):

Information as resource:

  • Clues accumulated (discovery resource)
  • Hypotheses formed (strategic resource)
  • Connections made (synthesis resource)
  • No depletion (all information permanent)

No traditional resources:

  • No health, stamina, items consumed
  • Pure information gathering
  • Supports Discovery aesthetic (no scarcity pressure)

Cognitive Load (Chapter 4):

Tracking load: High

  • 10-15 clues per area (4-6 areas per case)
  • Character names, relationships, timelines
  • Connections between clues
  • Mitigated by notebook system (offloads memory)

Decision load: Moderate

  • Which area to investigate next? (strategic)
  • Which connections to pursue? (hypothesis formation)
  • When to form hypothesis? (timing decision)
  • Not time-pressured (can think as long as needed)

Execution load: Very Low

  • Point-and-click interaction
  • Reading and examination
  • No timing windows
  • No reflexes required

Total cumulative load: High (tracking), but manageable

  • Appropriate for midcore audience ✓
  • Systems offload memory ✓
  • Time available for processing ✓
  • Deep engagement possible ✓

Balance (Chapter 6 - Forward Looking):

Paradigm: Revelation-Based

  • "Difficulty" = deduction complexity
  • Easy clues: Direct observations
  • Medium clues: Connections between two observations
  • Hard clues: Synthesis of many observations
  • Optional: Hint system reduces difficulty

Deduction complexity curve:

  • Early case: Straightforward connections (tutorial)
  • Mid cases: Multiple layers of deduction
  • Late cases: Complex synthesis required
  • Final case: Meta-synthesis of all previous cases

Player satisfaction:

  • "Aha!" moments = primary reward
  • Frequency: 2-3 minor revelations per area, 1 major per case
  • Emotional payoff: Story revelations tied to deductions
  • Skill demonstration: Solving hard cases without hints

Validation Plan

Phase 1: Micro Loop Testing (Week 1-2)

Test focus:

  • Observation actions satisfying?
  • Information presented clearly?
  • Clue gathering feels good?
  • Note-taking system intuitive?

Success criteria:

  • Players read/examine thoroughly (not skimming) ✓
  • Clues understood ✓
  • Notebook used naturally ✓
  • No confusion about observations ✓

Phase 2: Meso Loop Testing (Week 3-4)

Test focus:

  • Area investigation duration appropriate?
  • Hypothesis system clear?
  • Revelations satisfying?
  • Can solve without hints?

Success criteria:

  • Area investigation 30-50 minutes ✓
  • Hypothesis system understood ✓
  • Revelations feel earned ✓
  • 70%+ solve without hints ✓

Phase 3: Macro Loop Testing (Week 5-7)

Test focus:

  • Case duration appropriate for sessions?
  • Story arcs satisfying?
  • Stopping points clear?
  • Can return days later and resume?

Success criteria:

  • Case duration 2.5-4 hours ✓
  • Story closure satisfying ✓
  • Stopping points after revelations ✓
  • Return clarity excellent ✓

Phase 4: Meta Loop Testing (Week 8-12)

Test focus:

  • Sustained engagement across cases?
  • Meta-mystery intriguing?
  • Final synthesis satisfying?
  • Completionist content engaging?

Success criteria:

  • Complete multiple cases (80%+ finish 5+ cases) ✓
  • Meta-mystery interest high ✓
  • Final case synthesis payoff ✓
  • Optional content engaged (50%+) ✓

Expected Measurables

Duration Measurables:

  • Micro loop (observation): 1-4 minutes (player-controlled)
  • Meso loop (area): 30-50 minutes
  • Macro loop (case): 2.5-4 hours
  • Meta loop (all cases): 20-30 hours

Frequency Measurables:

  • Observations per area: 10-15
  • Areas per case: 4-6
  • Cases to completion: 8-10
  • Minor revelations per area: 2-3
  • Major revelations per case: 1

Pacing Measurables:

  • Intensity variance: Player-controlled (emergent)
  • Intensity peaks: During hypothesis formation
  • Calm periods: Reading, absorbing
  • No forced pacing

Stopping Point Measurables:

  • Major stopping points: Every 2.5-4 hours (after cases)
  • Secondary stopping points: Every 30-50 minutes (after areas)
  • Quality: 6/6 for major, 4/6 for secondary
  • Utilization: 80%+ exit at stopping points

Engagement Measurables:

  • Case completion rate: 80%+ finish 5+ cases
  • Discovery satisfaction: Very high ("aha!" moments)
  • Hint usage: 30% (70% solve independently)
  • Return rate: 70%+ return after 1 week

Design Outcomes Summary

"Midnight Archives" temporal architecture successfully:

✓ Serves Discovery aesthetic (investigation freedom, revelation payoff) ✓ Serves Narrative aesthetic (story revealed through deduction) ✓ Serves Challenge (light) aesthetic (deduction difficulty) ✓ Matches midcore audience capacity (2-4 hour sessions, high cognitive tolerance) ✓ Respects PC platform affordances (deep focus, mouse/keyboard precision) ✓ Provides excellent stopping points (after revelations, 6/6 elements) ✓ Enables emergent pacing (player controls depth) ✓ Offloads cognitive load appropriately (notebook system) ✓ No time pressure or FOMO (pure Discovery)

This demonstrates Discovery + Narrative + Challenge (light) for engaged midcore PC audience with player-respecting temporal architecture.

Case Study 3: "Iron Crown Tactics" - Tabletop RPG Campaign

Design Brief

Client Request: "We're a group of 5 friends who want to play a tactical fantasy campaign. We can meet weekly for 3-4 hours. We love tactical combat (X-COM, Fire Emblem) but want meaningful narrative between fights. Medium-to-high complexity acceptable. We're experienced with RPGs but tired of D&D 5e's combat taking forever."

Fixed Constraints:

  • Medium: Tabletop RPG (hybrid - battle maps for combat, theatre-of-mind for narrative)
  • Group: 5 players + 1 GM
  • Sessions: 3-4 hours weekly
  • Audience: Midcore-to-hardcore (experienced gamers, high tolerance for complexity)

Negotiable:

  • Exact combat mechanics
  • Campaign structure
  • Complexity level (as long as tactical depth present)

Design Freedom:

  • Complete system design
  • Temporal architecture
  • Campaign pacing

Aesthetic Analysis

Primary Aesthetic: Challenge

  • Tactical combat difficulty
  • Strategic decision-making
  • Mastery of combat systems
  • Victory through smart play

Secondary Aesthetic: Fellowship

  • Playing together weekly
  • Shared stories and victories
  • Table talk and banter
  • Social bonding through play

Tertiary Aesthetic: Narrative

  • Character development
  • Story progression
  • Emotional investment in outcomes
  • Campaign arc satisfaction

Audience Capacity:

  • Midcore-to-Hardcore: 3-4 hour sessions, high cognitive tolerance, experienced gamers
  • Weekly cadence (gaps between sessions)
  • Prefer tactical depth over simplicity
  • Want meaningful choices, not dice randomness

Target: Design for engaged tactical play with efficient session structure

Temporal Requirements Derivation

From Challenge aesthetic:

  • Combat should reward tactics, not luck
  • Decisions should matter (high stakes)
  • Learning curve across campaign
  • Mastery demonstration through performance

From Fellowship aesthetic:

  • Everyone needs spotlight time
  • Social time built into session
  • Coordination between players encouraged
  • No player dominates

From Narrative aesthetic:

  • Story progresses each session
  • Character growth visible
  • Emotional beats between combats
  • Campaign arc feels purposeful

From Midcore-to-Hardcore Audience:

  • Complex systems acceptable
  • 3-4 hour sessions standard
  • Weekly cadence (need recaps)
  • High cognitive load tolerated

From Tabletop Medium:

  • GM adjudication (flexibility)
  • Social table dynamics
  • Physical components (maps, tokens)
  • Theatre-of-mind possible

Synthesized Temporal Requirements:

  1. Combat encounters: 45-75 minutes maximum (not 2+ hours)
  2. Narrative scenes: 15-30 minutes (between combats)
  3. Session structure: 2-3 major beats
  4. Stopping points: Clear end of session
  5. Recap system: Essential for weekly gaps
  6. Spotlight distribution: Everyone gets moments

Loop Architecture Design

Micro Loop: 2-5 minutes (player turn)

Design Decision:

  • Player turn structure:
    • Movement: 10-30 seconds (move token on grid)
    • Action selection: 30-60 seconds (choose ability/attack)
    • Resolution: 30-60 seconds (roll, apply effects)
    • Other player reactions: 30-60 seconds (opportunity attacks, interrupts)
    • Total: 2-4 minutes per player turn

Rationale:

  • Faster than D&D 5e (turns often 5-10 minutes)
  • Tactical depth maintained through pre-planning
  • Simultaneous resolution where possible
  • GM adjudicates quickly (no rulebook diving)

Meso Loop: 45-75 minutes (combat encounter)

Design Decision:

  • Combat encounter structure:
    • Setup: 5 minutes (deploy tokens, explain situation)
    • Rounds: 6-10 rounds (5 players + enemies = ~8 minutes per round)
    • Resolution: 5-10 minutes (loot, aftermath, transition)
    • Total: 45-75 minutes

Rationale:

  • Complete encounter in single sitting
  • Multiple player turns per combat (4-6 turns each)
  • Enough rounds for tactics to matter
  • Fits within session budget (2-3 encounters possible)

Macro Loop: 3-4 hours (session)

Design Decision:

  • Session structure:
    • Recap: 10 minutes (what happened last week)
    • Opening scene: 15-20 minutes (narrative setup)
    • Combat encounter #1: 60 minutes
    • Narrative interlude: 15 minutes (character moments)
    • Combat encounter #2: 60 minutes
    • Closing scene: 15-20 minutes (consequences, setup next session)
    • Total: 3-3.5 hours

Rationale:

  • 2 major combats per session (satisfies Challenge aesthetic)
  • Narrative between combats (character development)
  • Clear stopping point (closing scene provides closure)
  • Predictable structure (players know what to expect)

Meta Loop: 20-30 sessions (campaign)

Design Decision:

  • Campaign structure:
    • Act 1: Sessions 1-8 (introduce threat)
    • Act 2: Sessions 9-18 (escalation and complications)
    • Act 3: Sessions 19-25 (climax and resolution)
    • Epilogue: Session 26-28
    • Total: 26-28 sessions (6-7 months weekly)

Rationale:

  • Manageable commitment (half year)
  • Clear narrative arc
  • Multiple story beats across acts
  • Satisfying conclusion without endless play

Between-Session Gap: 1 week

Design Decision:

  • Mandatory recap at session start
  • GM provides summary document via email
  • Players can review character sheets between sessions
  • No homework required (but welcomed)

Rationale:

  • Weekly gaps = memory loss
  • Recap essential for continuity
  • Optional prep for engaged players
  • Respects everyone's time

Pacing Pattern Selection

Primary: Episodic

Implementation:

  • Each session = complete episode
  • Clear beginning (recap, setup)
  • Middle (encounters and narrative)
  • End (resolution, consequences)
  • Cliffhanger optional but light

Why this pattern:

  • Fellowship benefits from episodic (shared complete experiences)
  • Challenge aesthetic needs closure (victories feel complete)
  • Weekly cadence fits episodes perfectly
  • Can miss session without losing continuity

Secondary: Escalating (within encounters)

  • Encounter difficulty increases throughout
  • Combat intensity builds to climax
  • Each round stakes higher
  • Victory earned through escalation

Stopping Point Design

Primary Stopping Point: End of every session (3-4 hours)

Six elements check:

  1. Clear completion signal: ✓

    • Closing scene provides narrative beat
    • GM says "and that's where we'll stop"
    • Recap of session events
    • Preview next session

  2. Progress persistence: ✓

    • Character sheets updated
    • Campaign notes maintained
    • Story progress clear
    • XP/loot distributed

  3. State simplification: ✓

    • Session ends between encounters (not mid-combat)
    • Characters in safe location typically
    • No ongoing immediate threats
    • Clean break point

  4. Emotional resolution: ✓

    • Closing scene provides emotional beat
    • Victories acknowledged
    • Losses mourned
    • Next steps implied

  5. Re-entry clarity: ✓

    • Recap email sent by GM
    • Character sheets accessible
    • Story summary available
    • Next session goal clear

  6. Time visibility: ✓

    • Session structure known (3-4 hours)
    • Can see progress through encounters
    • GM often says "we have time for one more scene"
    • Explicit time management

Result: Excellent stopping points at session end (weekly)

Secondary Stopping Point: Between encounters (every 60-90 minutes)

Partial elements:

  • Completion signal: ✓ (encounter victory)
  • Progress persistence: ✓ (all tracked)
  • State simplification: ✓ (between combats)
  • Emotional resolution: Partial (minor)
  • Re-entry clarity: ✓ (mid-session)
  • Time visibility: Partial (GM manages)

Result: Good mid-session break points (bio breaks, snack)

Integration Analysis

Resources (Chapter 3):

Character resources:

  • Health: Depletes in combat, recovers between encounters
  • Abilities: Limited uses per encounter (resource management)
  • Consumables: Potions, scrolls (tactical choices)
  • Recovery rates: Fast (supports multiple encounters per session)

Campaign resources:

  • Gold: Earned gradually, spent on upgrades
  • Reputation: Narrative currency (affects story)
  • Time pressure: Optional (GM can add urgency)

Resource design:

  • Supports Challenge (tactical resource management)
  • Fast recovery enables 2-3 combats per session
  • No grinding needed (respect player time)

Cognitive Load (Chapter 4):

Tracking load: High

  • Character abilities (5-10 per character)
  • Enemy positions and stats
  • Terrain effects
  • Ongoing conditions
  • BUT: Character sheets offload most of this
  • Map shows positions visually

Decision load: High

  • Player turn: Multiple viable actions
  • Positioning critical
  • Ability selection meaningful
  • Coordination with party

Execution load: Low

  • Turn-based (no time pressure)
  • Rolling dice simple
  • Moving token simple
  • Complexity in decision, not execution

Total cumulative load: High (appropriate for midcore-hardcore)

  • Complexity welcomed by audience ✓
  • Tactical depth achieved ✓
  • Social time provides cognitive breaks ✓
  • GM manages complexity ✓

Balance (Chapter 6 - Forward Looking):

Paradigm: Consistency-Based

  • Enemy abilities telegraphed
  • Rules consistent (learnable)
  • Player skill matters (not dice luck)
  • Encounters designed for difficulty curve

Encounter difficulty:

  • Session 1-4: Learning (medium difficulty)
  • Session 5-12: Mastery developing (medium-hard)
  • Session 13-20: Expert play (hard)
  • Session 21-28: Climax challenges (very hard but fair)

Party coordination:

  • Synergies between character abilities
  • Tactics emerge from coordination
  • Fellowship aesthetic supported by tactical cooperation
  • Social table talk = strategic planning

Phase 1: Encounter Testing (Sessions 1-3)

Test focus:

  • Encounter duration appropriate?
  • Turns taking reasonable time?
  • Tactical depth satisfying?
  • Everyone getting spotlight?

Success criteria:

  • Encounters 45-75 minutes ✓
  • Player turns 2-5 minutes ✓
  • Tactical satisfaction high ✓
  • Spotlight distribution even ✓

Phase 2: Session Structure Testing (Sessions 4-8)

Test focus:

  • Session structure working?
  • Pacing between combat and narrative good?
  • Stopping points satisfying?
  • Recaps effective?

Success criteria:

  • Session duration 3-4 hours ✓
  • 2 combats + narrative works ✓
  • Closing scenes provide closure ✓
  • Weekly recaps maintain continuity ✓

Phase 3: Campaign Arc Testing (Sessions 9-16)

Test focus:

  • Story progression satisfying?
  • Character growth visible?
  • Difficulty curve appropriate?
  • Still engaged mid-campaign?

Success criteria:

  • Narrative engagement high ✓
  • Character advancement satisfying ✓
  • Encounters appropriately challenging ✓
  • Attendance maintained (Fellowship) ✓

Test focus:

  • Climax satisfying?
  • Resolution complete?
  • Would play another campaign?

Success criteria:

  • Final encounters epic ✓
  • Story closure satisfying ✓
  • Group wants to continue (new campaign) ✓

Duration Measurables:

  • Micro loop (player turn): 2-5 minutes
  • Meso loop (encounter): 45-75 minutes
  • Macro loop (session): 3-4 hours
  • Meta loop (campaign): 26-28 sessions (6-7 months)

Frequency Measurables:

  • Player turns per encounter: 4-6
  • Encounters per session: 2 (occasionally 3)
  • Sessions per month: 4 (weekly)
  • Campaign duration: 26-28 weeks

Pacing Measurables:

  • Episodic structure: Clear per session
  • Escalating intensity: Within encounters
  • Narrative breaks: Between encounters
  • Social time: Throughout session

Stopping Point Measurables:

  • Primary stopping points: Weekly (end of session)
  • Quality: 6/6 elements
  • Secondary breaks: Between encounters (break time)
  • Session overruns: Rare (<10 minutes)

Engagement Measurables:

  • Attendance: 90%+ (5 players, rare absences)
  • Return rate: 100% (committed group)
  • Satisfaction: High (post-session feedback)
  • Campaign completion: Expected 100%

Design Outcomes Summary

"Iron Crown Tactics" temporal architecture successfully:

✓ Serves Challenge aesthetic (tactical combat, meaningful decisions) ✓ Serves Fellowship aesthetic (weekly group engagement, coordination) ✓ Serves Narrative aesthetic (story progression, character growth) ✓ Matches midcore-hardcore audience (3-4 hour sessions, high complexity) ✓ Respects tabletop medium (GM flexibility, social dynamics) ✓ Provides excellent stopping points (weekly session ends, 6/6 elements) ✓ Efficient encounter design (45-75 minutes vs. D&D's 2+ hours) ✓ Enables episodic structure (weekly cadence) ✓ Supports group coordination (Fellowship + Challenge synergy)

This demonstrates Challenge + Fellowship + Narrative for engaged tabletop group with efficient temporal architecture that respects everyone's time.

Key Learnings

1. Same aesthetic, different implementations:

  • Challenge in Neon Vanguard: Light, positioning-based, automatic combat
  • Challenge in Iron Crown Tactics: Deep, tactical, turn-based combat
  • Both valid Challenge scaffolds for different audiences

2. Platform dictates constraints:

  • Mobile: Short sessions, one-hand, interruptions
  • PC: Long sessions, deep focus, complex interaction
  • Tabletop: Social dynamics, GM flexibility, group coordination

3. Stopping points adapt to context:

  • Mobile: Every run (12 minutes) - frequent, predictable
  • PC: After revelations (2.5-4 hours) - substantial, earned
  • Tabletop: Weekly sessions - social commitment, group schedule

4. Cognitive load matches capacity:

  • Casual: Very low (track 4 elements)
  • Midcore: High but offloaded (notebook systems)
  • Midcore-Hardcore: Very high (10+ abilities, tactical complexity)

5. Pacing serves aesthetic:

  • Escalating for Challenge (Neon Vanguard)
  • Emergent for Discovery (Midnight Archives)
  • Episodic for Fellowship (Iron Crown Tactics)

Conclusion

These three case studies demonstrate:

  1. Complete derivation process from aesthetic goals to specific temporal parameters
  2. Different aesthetic/audience/platform combinations require completely different temporal architectures
  3. Integration with resources, cognitive load, and balance is essential for coherent design
  4. Stopping points must adapt to platform and audience capacity
  5. Validation plans specific to each context ensure temporal architecture works

Use these case studies as templates:

  • Follow the derivation process for your own games
  • Adapt patterns to your specific constraints
  • Validate with measurables appropriate to your context
  • Iterate based on testing data

Remember: These are invented games demonstrating principles. Your game will have different constraints, different aesthetics, different audiences. Always derive your temporal architecture from YOUR specific goals and constraints.

The framework works across all contexts, but the specific implementation must serve your unique purpose.
