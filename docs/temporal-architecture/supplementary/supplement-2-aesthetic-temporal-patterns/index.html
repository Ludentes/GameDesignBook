Supplement 2: Aesthetic-Temporal Pattern Catalog

Purpose: Comprehensive reference for aesthetic→temporal architecture mappings with multiple examples per aesthetic

This supplement provides:

  • Extended patterns for all common aesthetics
  • Multiple implementation examples per aesthetic
  • Edge cases and hybrid aesthetics
  • Player capacity variations detailed
  • When standard patterns don't apply
  • Cross-aesthetic combinations

How to Use This Catalog

This is a reference document, not a tutorial. Each aesthetic pattern includes:

  1. Temporal requirements: What properties time structure must have
  2. Loop architecture: Standard durations for each scale
  3. Pacing pattern: Which of the five patterns fits best
  4. Player capacity variants: How pattern changes for casual/midcore/hardcore
  5. Multiple examples: Games demonstrating the pattern
  6. Common mistakes: Where designs typically fail
  7. Integration notes: Resources, cognitive load, balance considerations

Use this to:

  • Look up appropriate temporal structure for your aesthetic
  • See how successful games implement patterns
  • Understand variations based on player capacity
  • Avoid common mistakes

Don't use this to:

  • Copy exact numbers blindly (context matters)
  • Ignore your specific constraints
  • Skip the derivation process (understand WHY)

Pattern Format

Each aesthetic follows this structure:

Challenge Aesthetic

Core Experience: Overcoming difficulty through skill development and persistence. Players want to fail, learn, and eventually succeed through demonstrated mastery.

Temporal Requirements

Fast iteration:

  • Pattern learning requires repeated attempts
  • Long delays between attempts disrupt learning
  • Must retry immediately after failure

Clear feedback loops:

  • Success/failure must be instantly obvious
  • Player must understand WHY they failed
  • Cause-and-effect relationships must be clear

Learnable consistency:

  • Same inputs produce same outputs (deterministic or telegraphed)
  • Patterns exist to be learned
  • Randomness minimized or clearly communicated

Mastery demonstration:

  • Difficulty doesn't disappear when learned
  • Skill ceiling allows continued improvement
  • Speed, efficiency, or style optimization possible

Appropriate session length:

  • Enough time for multiple learning attempts
  • Not so long that single session exhausts player
  • Clear stopping points after victories

Standard Loop Architecture

Micro loops: 5-15 seconds

  • Action → Feedback cycle must be tight
  • Low complexity per action (can iterate rapidly)
  • Execution becomes automatic with mastery

Meso loops: 2-5 minutes

  • Full challenge attempt start → complete
  • Multiple micro loops chained
  • 10-20 attempts reasonable per session
  • Clear success/failure endpoint

Macro loops: 45-90 minutes

  • Multiple meso challenges with progression
  • Session provides complete learning arc
  • Stopping points after major victories
  • Visible skill improvement across session

Meta loops: 10-50 hours

  • Mastery development across sessions
  • Optional (core satisfaction from meso/macro)
  • When present: Unlocks, challenges, rankings

Failure recovery: <10 seconds

  • Death → retry essentially instant
  • Minimal progress loss
  • No punishment beyond time investment

Pacing Pattern: Escalating or Episodic

Escalating (within meso/macro):

  • Difficulty builds within each challenge
  • Intensity climbs to climax
  • Resolution provides release
  • Reset for next challenge

Episodic (across session):

  • Each challenge is complete episode
  • Clear beginning, middle, end
  • Progress between episodes
  • Next episode fresh difficulty curve

Player Capacity Variants

Casual Players (5-30 minute sessions):

  • Micro loops: 5-10 seconds (very tight)
  • Meso loops: 1-3 minutes (very short challenges)
  • Macro loops: 10-20 minutes (multiple quick challenges)
  • Meta loops: Optional or absent
  • Failure recovery: <5 seconds (instant)
  • Example: Mobile action games, Wordle

Midcore Players (30-90 minute sessions):

  • Micro loops: 5-15 seconds (tight)
  • Meso loops: 3-5 minutes (learnable challenges)
  • Macro loops: 60-90 minutes (multiple challenges)
  • Meta loops: Present, moderate depth
  • Failure recovery: <10 seconds
  • Example: Celeste, Hades, Slay the Spire

Hardcore Players (2+ hour sessions):

  • Micro loops: 10-20 seconds (can be more complex)
  • Meso loops: 5-15 minutes (deeper challenges)
  • Macro loops: 2-6 hours (interconnected sequences)
  • Meta loops: Deep, extensive (50-200+ hours)
  • Failure recovery: <15 seconds (can be slightly longer)
  • Example: Dark Souls, Elden Ring, Hollow Knight

Examples

Example 1: Celeste (Midcore Challenge)

Temporal architecture:

  • Micro: 3-5 seconds (jump → dash → grab sequence)
  • Meso: 2-4 minutes per screen (20-40 screens per chapter)
  • Macro: 45-90 minutes per chapter
  • Meta: 15-30 hours (complete game, B-sides, C-sides)
  • Failure recovery: <1 second (instant respawn)

Why it works:

  • Instant failure recovery enables rapid iteration (pattern learning)
  • Screen length perfect for learning curve (2-4 minutes = 15-30 attempts)
  • Chapters provide session structure with clear stopping points
  • Micro-loop mastery feels amazing (precision movement)
  • Meta challenges (B-sides, C-sides) reward mastery development

Measurables observed:

  • Micro cycle: 3-5 seconds ✓
  • Meso completion: 2-4 minutes ✓
  • Attempts per hour: 15-25 ✓ (high iteration)
  • Sessions to chapter mastery: 2-5 ✓
  • Failure recovery: <1 second ✓

Example 2: Vampire Survivors (Casual Challenge)

Temporal architecture:

  • Micro: 5-10 seconds (positioning decisions)
  • Meso: 30-60 seconds (until next level-up choice)
  • Macro: 30 minutes (full run)
  • Meta: 10-20 hours (unlock all characters/weapons)
  • Failure recovery: 5 seconds (restart run)

Why it works:

  • Extremely simple micro (just movement, combat automatic)
  • Frequent meso payoffs (level-up every 30-60s)
  • Fixed macro duration (always 30 minutes, visible timer)
  • Casual-appropriate complexity (low cognitive load)
  • Perfect stopping point (run complete or failure = exit)

Measurables observed:

  • Micro cycle: 5-10 seconds ✓
  • Meso (level-up): 30-60 seconds ✓
  • Macro (run): 30 minutes (fixed) ✓
  • Meta completion: 10-20 hours ✓
  • Casual accessibility: High ✓

Example 3: Super Meat Boy (Hardcore Challenge)

Temporal architecture:

  • Micro: 2-4 seconds (very fast platforming)
  • Meso: 15-45 seconds per level (very short but very hard)
  • Macro: 60-120 minutes (world completion)
  • Meta: 30-80 hours (Dark World, bandages, A+ ranks)
  • Failure recovery: <1 second (instant restart)

Why it works:

  • Brutally fast micro-loop (hardcore execution requirement)
  • Very short meso but extremely difficult (100+ attempts common)
  • Instant retry maintains flow despite high death count
  • Meta challenges expand depth infinitely for hardcore audience
  • Hardcore difficulty appropriate for capacity

Measurables observed:

  • Micro cycle: 2-4 seconds ✓
  • Meso completion: 15-45 seconds ✓
  • Attempts per hour: 40-80+ ✓ (very high iteration)
  • Sessions to level mastery: 5-15 ✓
  • Hardcore difficulty: Appropriate for audience ✓

Common Mistakes

Mistake 1: Slow failure recovery

  • Recovery > 10 seconds breaks learning flow
  • Players lose momentum between attempts
  • Frustration from wasted time, not from challenge
  • Solution: Instant respawn, minimal animations

Mistake 2: Unclear failure causes

  • Players can't learn from attempts
  • Success feels random, not earned
  • No skill development possible
  • Solution: Clear telegraphs, consistent rules

Mistake 3: Meso loops too long

  • 10+ minute attempts = too much to learn at once
  • Players forget early-attempt learnings by end
  • Discouraging to restart long attempts
  • Solution: Break into shorter challenges with checkpoints

Mistake 4: No mastery demonstration

  • Challenge disappears once learned
  • Nothing to optimize or improve
  • No reason to replay
  • Solution: Add speed-run potential, scoring, style points

Mistake 5: Poor stopping points

  • No clear victories to exit after
  • "One more challenge" trap
  • Players binge from inability to stop
  • Solution: Chapter structure, clear completions

Integration Notes

Resources (Chapter 3):

  • Health/lives should enable learning (not punish failure excessively)
  • Resource scarcity can create additional challenge layer
  • Recovery rate must match failure frequency
  • Example: Celeste has no resource depletion, pure skill test

Cognitive Load (Chapter 4):

  • Micro-loop load must decrease with mastery (automaticity essential)
  • Total cumulative load across scales must match player capacity
  • Casual: Very low micro, simple meso
  • Hardcore: Can handle complex micro, deep meso

Balance (Chapter 6):

  • TTK (Time To Kill) must fit meso loop duration
  • Difficulty must enable learning curve (not too easy, not impossible)
  • Pattern consistency critical (consistency paradigm from Balance chapter)

Discovery Aesthetic

Core Experience: Finding and learning new things, uncovering secrets, making connections between observations. Players want revelation through exploration and investigation.

Temporal Requirements

Exploration freedom:

  • Cannot be rushed (players need time to observe)
  • Must allow backtracking and re-examination
  • Player controls pacing entirely

Pattern recognition space:

  • Time to notice details
  • Time to form hypotheses
  • Time to test theories
  • Multiple observations required for understanding

Memory accumulation:

  • Discoveries persist across sessions
  • Mental models build over time
  • No time pressure to "use" discoveries

Revelation moments:

  • Clear feedback when connection made
  • Satisfying "aha!" experience
  • Visible progress in understanding

Variable pacing:

  • Some moments calm (observation)
  • Some moments intense (active problem-solving)
  • Player controls transitions

Standard Loop Architecture

Micro loops: 30 seconds - 5 minutes

  • Observation actions (examine, read, listen)
  • Highly variable duration based on content
  • No time pressure on individual actions
  • Cognitive load in processing, not execution

Meso loops: 10-40 minutes

  • Investigation of specific area or mystery element
  • Multiple observations accumulate
  • Hypothesis formation and testing
  • Revelation or progress toward understanding

Macro loops: 2-6 hours

  • Major mystery progression
  • Multiple meso loops connected
  • Significant revelation or discovery
  • Natural stopping after major "aha" moment

Meta loops: 15-50 hours

  • Complete understanding of game's mysteries
  • Connections between all discoveries
  • Final revelations require synthesis
  • Mastery = comprehensive knowledge

Pacing Pattern: Emergent or Episodic

Emergent (player-controlled):

  • Player decides when to investigate deeply vs. move on
  • Intensity varies based on player engagement
  • No forced pacing
  • Supports different play styles

Episodic (when structured):

  • Clear investigation arcs per episode
  • Each episode has discovery payoff
  • Can pause between episodes
  • Outer Wilds uses this (20-minute loop episodes)

Player Capacity Variants

Casual Players (5-30 minute sessions):

  • Micro loops: 30 seconds - 2 minutes (simple observations)
  • Meso loops: 10-15 minutes (small mystery complete)
  • Macro loops: 20-40 minutes (session = one investigation)
  • Meta loops: Optional or very light
  • Example: Wordle, simple puzzle games

Midcore Players (30-90 minute sessions):

  • Micro loops: 1-3 minutes (detailed observations)
  • Meso loops: 20-30 minutes (investigation sequences)
  • Macro loops: 90-150 minutes (major discovery)
  • Meta loops: Present, moderate depth
  • Example: Outer Wilds, Return of the Obra Dinn

Hardcore Players (2+ hour sessions):

  • Micro loops: 2-5 minutes (deep investigation)
  • Meso loops: 30-60 minutes (complex investigations)
  • Macro loops: 3-8 hours (interconnected mysteries)
  • Meta loops: Deep, extensive
  • Example: La-Mulana, Fez (cryptographic level)

Examples

Temporal architecture:

  • Micro: 30 seconds - 3 minutes (read log, observe event, explore location)

Why it works:

  • 20-minute loop provides structure without pressure
  • Each loop = exploration + one new thing learned
  • Ship log persistent (offloads memory)
  • Revelations earned through connecting observations
  • Player controls what to investigate and when

Measurables observed:

  • Micro: Highly variable (player-controlled) ✓

  • Macro: 3-5 hours per major discovery ✓
  • Meta: 20-30 hours ✓
  • No time pressure: Players confirm ✓

Common Mistakes

Mistake 1: Time pressure

  • Forced pacing destroys observation time
  • Players rush, miss clues
  • Discovery requires thinking time
  • Solution: Never time-limit investigations

Mistake 2: Unclear progress

  • Players don't know what they've learned
  • Can't see advancement toward understanding
  • Wandering without purpose
  • Solution: Ship logs, notebooks, progress indicators

Mistake 3: Memory overload

  • Too many details to remember
  • No system to offload memory
  • Players forget connections
  • Solution: Persistent logs, relationship maps

Mistake 4: No revelation feedback

  • Connections made but not acknowledged
  • Players unsure if understanding correct
  • No "aha!" moment satisfaction
  • Solution: Explicit confirmation when discoveries made

Mistake 5: Too linear

  • Only one investigation path
  • No emergent discovery
  • Feels scripted, not explored
  • Solution: Multiple valid approaches, non-linear

Integration Notes

Resources (Chapter 3):

  • Typically no resource depletion (nothing to "spend")
  • Information is the resource (accumulated knowledge)
  • No scarcity pressure (can investigate freely)

Cognitive Load (Chapter 4):

  • High tracking load (many observations to remember)
  • Mitigate with persistent logs and organization tools
  • Decision load moderate (which thread to follow?)
  • Execution load very low (no reflexes needed)

Balance (Chapter 6):

  • "Difficulty" is about connection complexity, not execution
  • Must have enough clues (not too sparse)
  • But not so obvious no deduction needed
  • Multiple difficulty paths (some easy discoveries, some hard)

Mastery Aesthetic

Core Experience: Deep system understanding and endless optimization. Players want to learn systems thoroughly and discover optimal strategies.

Temporal Requirements

Long optimization windows:

  • Time to experiment with builds/strategies
  • Space for testing hypotheses
  • No pressure to move on quickly

Consistent iteration:

  • Practice makes perfect
  • Regular engagement enables learning
  • Feedback on optimization attempts

Escalating complexity:

  • Challenge grows with skill
  • Always more to learn/optimize
  • Systems reveal depth over time

Meta-progression:

  • Learning carries across sessions
  • Mastery development visible
  • Community knowledge sharing

Standard Loop Architecture

Micro loops: 10-30 seconds

  • Complex enough to optimize
  • Allow meaningful decision-making
  • Skill expression possible

Meso loops: 15-60 minutes

  • Build testing and refinement
  • Encounter variety to validate builds
  • Clear success/failure metrics

Macro loops: 2-6 hours

  • Deep strategy development
  • Multiple build iterations per session
  • Major progression milestones

Meta loops: 50-500+ hours

  • Complete system mastery
  • Community meta-game
  • Seasonal resets or expansions

Pacing Pattern: Steady-State or Emergent

Steady-state:

  • Consistent engagement level
  • Flow state from optimization
  • Regular practice rhythm

Emergent:

  • Player controls intensity
  • Can chase optimization or relax
  • Community-driven pacing

Player Capacity Variants

Casual Players:

  • Mastery aesthetic difficult for casual
  • If attempted: Very simple systems, visible optimization
  • Short optimization windows (5-15 minutes)
  • Example: Tetris effect optimization

Midcore Players (30-90 minute sessions):

  • Micro: 10-20 seconds
  • Meso: 20-45 minutes (build testing)
  • Macro: 60-120 minutes (strategy development)
  • Meta: 30-100 hours
  • Example: Slay the Spire, Into the Breach

Hardcore Players (2+ hour sessions):

  • Micro: 15-30 seconds (complex decisions)
  • Meso: 30-90 minutes (deep optimization)
  • Macro: 3-8 hours (complete strategy)
  • Meta: 100-1000+ hours
  • Example: Path of Exile, Factorio, StarCraft II

Common Mistakes

Mistake 1: Shallow depth

  • Optimal strategy obvious quickly
  • Nothing left to learn
  • Players master and leave
  • Solution: Layered systems, emergent complexity

Mistake 2: No feedback on optimization

  • Can't tell if build is good
  • No metrics to compare
  • Optimization feels arbitrary
  • Solution: Clear performance metrics, testing environments

Mistake 3: Casual audience targeting

  • Mastery requires time investment
  • Casual capacity insufficient
  • Frustration from complexity
  • Solution: Either simplify drastically or retarget audience

Mistake 4: Static meta

  • Game doesn't evolve
  • Community solves, no new challenges
  • Staleness after first meta-game solved
  • Solution: Seasonal content, balance patches, expansions

Mistake 5: Poor community tools

  • No build sharing
  • No strategy discussion platforms
  • Mastery development isolated
  • Solution: In-game sharing, external community support

Integration Notes

Resources (Chapter 3):

  • Complex resource economies enable optimization
  • Multiple resource types create trade-offs
  • Resource efficiency = optimization target

Cognitive Load (Chapter 4):

  • Very high load appropriate for Mastery
  • Hardcore audience wants this complexity
  • Load should decrease at micro scale with practice
  • Meta-level load can remain high indefinitely

Balance (Chapter 6):

  • Multiple viable strategies required (not one optimal)
  • Skill ceiling extremely high
  • Performance measurables essential
  • Regular rebalancing maintains meta freshness

Tension Aesthetic

Core Experience: Sustained pressure, high stakes, constant danger. Players want to feel on edge, where every decision matters.

Temporal Requirements

Sustained pressure:

  • Can't fully relax at any point
  • Stakes maintained throughout
  • Consequences feel significant

Manageable intensity:

  • Can't exhaust players completely
  • Need occasional relief beats
  • Pacing prevents burnout

High stakes maintained:

  • Decisions have lasting consequences
  • Failure is costly (but not game-ending)
  • Success feels earned

Strategic relief moments:

  • Brief recovery without breaking tension
  • Planning phases between action
  • Calm before next storm

Standard Loop Architecture

Micro loops: Continuous

  • Always something happening
  • No complete downtime
  • Attention required constantly

Meso loops: 15-35 minutes

  • Sustained intensity manageable at this scale
  • Mission/encounter with beginning and end
  • Relief after completion

Macro loops: 60-120 minutes

  • Multiple meso loops with cumulative stakes
  • Bounded to prevent exhaustion
  • Clear session endpoints

Relief beats: Every 15-25 minutes

  • Brief moments of lower intensity
  • Planning, preparation, recovery
  • Not complete safety, just reduced pressure

Pacing Pattern: Escalating or Sustained

Escalating (within meso):

  • Intensity builds within encounter
  • Peaks at climax
  • Resolution provides relief

Sustained (across macro):

  • Baseline tension never fully releases
  • Meta-pressure (campaign timers, resource depletion)
  • Stakes accumulate

Player Capacity Variants

Casual Players:

  • Tension aesthetic difficult for casual
  • If attempted: Very short bursts (5-10 minutes max)
  • Clear safe zones
  • Example: Short horror games

Midcore Players (30-90 minute sessions):

  • Micro: Continuous (with brief respites)
  • Meso: 20-30 minutes (mission structure)
  • Macro: 60-90 minutes (bounded intensity)
  • Relief: Every 20-30 minutes
  • Example: XCOM 2, Alien: Isolation (shorter sessions)

Hardcore Players (2+ hour sessions):

  • Micro: Continuous
  • Meso: 30-60 minutes (longer encounters)
  • Macro: 2-4 hours (campaign segments)
  • Relief: Every 30-45 minutes
  • Example: XCOM 2 Ironman, Darkest Dungeon

Common Mistakes

Mistake 1: No relief beats

  • Exhausts players
  • Can't sustain tension for hours
  • Burnout inevitable
  • Solution: Plan relief phases every 15-30 minutes

Mistake 2: False tension

  • Stakes don't actually matter
  • Consequences easily reversed
  • Players realize it's safe
  • Solution: Meaningful stakes (permadeath, resources lost)

Mistake 3: Too long

  • Tension can't last 4+ hours
  • Diminishing returns on stress
  • Players disengage to cope
  • Solution: Bound sessions to 60-120 minutes

Mistake 4: Casual audience targeting

  • Tension requires sustained attention
  • Casual players seeking relaxation
  • Mismatch in goals
  • Solution: Target midcore/hardcore or reduce intensity

Mistake 5: Unclear threats

  • Don't know what's dangerous
  • Can't make meaningful decisions
  • Anxiety without agency
  • Solution: Telegraph threats clearly

Integration Notes

Resources (Chapter 3):

  • Scarcity critical for Tension
  • Limited health, ammo, resources
  • Recovery rates slow (pressure maintenance)
  • Depletion creates stakes

Cognitive Load (Chapter 4):

  • Moderate load appropriate
  • Too high = overwhelms
  • Too low = not engaging
  • Tracking: Limited resources
  • Decisions: Risk assessment
  • Execution: Moderate

Balance (Chapter 6):

  • Difficult but fair
  • Mistakes costly but not game-ending
  • Skill rewarded
  • Risk/reward paradigm

Submission Aesthetic

Core Experience: Relaxation, flow state, "turning brain off." Players want gentle, pleasant engagement without stress.

Temporal Requirements

Flow state maintenance:

  • Rhythmic, predictable patterns
  • No sudden difficulty spikes
  • Smooth execution

No pressure spikes:

  • No time limits
  • No high-stakes decisions
  • Failures gentle

Flexible stopping points:

  • Can exit anytime
  • No loss for stopping
  • Easy to resume

Gentle progression:

  • Advancement without stress
  • Visible progress
  • No competitive pressure

Standard Loop Architecture

Micro loops: 5-15 seconds

  • Smooth, low-friction execution
  • Simple, comfortable actions
  • Automatic with minimal thought

Meso loops: 10-30 minutes

  • Gentle goals, no pressure
  • Progress without urgency
  • Satisfying but not stressful

Macro loops: 30-120 minutes (or infinite)

  • Player decides when to stop
  • No forced endpoints
  • Sessions any length comfortable

Meta loops: Optional

  • Progression not required
  • Can engage deeply or casually
  • No FOMO, no obligations

Pacing Pattern: Cyclical or Steady-State

Cyclical:

  • Regular, comfortable rhythm
  • Predictable oscillations
  • Meditative quality

Steady-state:

  • Consistent gentle engagement
  • Flow state
  • Timeless quality

Player Capacity Variants

All player types work well:

  • Submission accessible to everyone
  • Duration and depth vary

Casual (5-30 minutes):

  • Micro: 5-10 seconds
  • Meso: 10-20 minutes
  • Macro: 15-30 minutes
  • Example: Mobile puzzle games, casual sims

Midcore (30-90 minutes):

  • Micro: 5-15 seconds
  • Meso: 15-30 minutes
  • Macro: 30-90 minutes
  • Example: Stardew Valley, Animal Crossing

Hardcore (2+ hours):

  • Micro: 10-15 seconds
  • Meso: 20-40 minutes
  • Macro: 90-300+ minutes
  • Example: Euro Truck Simulator 2, Satisfactory (zen mode)

Examples

Example 1: Stardew Valley (Midcore Submission)

Temporal architecture:

  • Micro: 5-15 seconds (plant seed, water crop, talk to NPC)
  • Meso: 12-18 minutes (one in-game day)
  • Macro: 60-120 minutes (season or personal goal)
  • Meta: 50-200+ hours (farm completion, relationships)
  • Cyclical pacing (daily rhythm)

Why it works:

  • Daily cycle = predictable rhythm
  • No time pressure (can do anything or nothing)
  • Gentle progression (crops grow, relationships develop)
  • Can stop after any day
  • Relaxing activities

Measurables observed:

  • Cyclical pattern clear ✓
  • No pressure: Player-controlled ✓
  • Stopping points: Every day ✓
  • Submission satisfaction: High ✓

Example 2: Animal Crossing: New Horizons (Casual-Midcore Submission)

Temporal architecture:

  • Micro: 5-10 seconds (catch bug, pick fruit, talk to villager)
  • Meso: 15-30 minutes (daily tasks)
  • Macro: 20-60 minutes (personal session goal)
  • Meta: 100-500+ hours (island completion)
  • Real-time clock (cyclical pacing)

Why it works:

  • Real-time prevents binging
  • Daily routine comfortable
  • No penalties for skipping days
  • Gentle activities
  • Social without pressure

Measurables observed:

  • Daily engagement sustainable ✓
  • No FOMO (confirmed by design) ✓
  • Casual-appropriate ✓
  • Long-term engagement: High ✓

Example 3: Vampire Survivors (Casual Submission)

Temporal architecture:

  • Micro: 5-10 seconds (positioning only)
  • Meso: 30-60 seconds (level-up choices)
  • Macro: 30 minutes (fixed run duration)
  • Meta: 10-20 hours (all unlocks)
  • Steady-state pacing

Why it works:

  • Combat automatic (no execution stress)
  • Simple decisions (3 choices per level-up)
  • Fixed duration (visible timer)
  • Can win or fail comfortably
  • Perfect casual session length

Measurables observed:

  • Execution load: Very low ✓
  • Decision load: Low ✓
  • Session duration: 30 minutes fixed ✓
  • Casual accessibility: Excellent ✓

Common Mistakes

Mistake 1: Hidden pressure

  • "Relaxing" but has time limits
  • "Casual" but competitive leaderboards
  • "Chill" but resource scarcity stress
  • Solution: Remove all pressure sources

Mistake 2: Unclear progression

  • Can't see advancement
  • Feels aimless
  • Boredom from lack of structure
  • Solution: Visible, gentle progression

Mistake 3: Friction in micro-loops

  • Actions require precision
  • Timing windows strict
  • Cognitive demand high
  • Solution: Very forgiving execution

Mistake 4: No stopping points

  • Traps players like Challenge games
  • "One more day" problem
  • Exhaustion instead of relaxation
  • Solution: Clear session structures

Mistake 5: Unexpected difficulty spikes

  • Sudden hard challenges
  • Failure punishments
  • Stress introduction
  • Solution: Maintain gentle difficulty curve

Integration Notes

Resources (Chapter 3):

  • Abundant resources
  • No scarcity pressure
  • Recovery fast and comfortable
  • Can't "run out"

Cognitive Load (Chapter 4):

  • Very low across all dimensions
  • Tracking: Minimal
  • Decisions: Simple, low-stakes
  • Execution: Forgiving, automatic

Balance (Chapter 6):

  • Very easy difficulty
  • Failure gentle or impossible
  • No optimization pressure
  • Success paradigm (can't really fail)

Fellowship Aesthetic

Core Experience: Social connection and collaboration. Players want to engage with others meaningfully.

Temporal Requirements

Synchronization points:

  • Players need to coordinate
  • Simultaneous engagement
  • Shared timeline

Shared experiences:

  • Activities done together
  • Moments witnessed by all
  • Collective achievement

Communication windows:

  • Time to discuss and plan
  • Social interaction space
  • Not pure mechanical execution

Commitment clarity:

  • Respect others' time
  • Clear session expectations
  • Fair to all participants

Standard Loop Architecture

Micro loops: 5-30 seconds

  • Synchronous actions (together)
  • Turn-based or real-time coordination
  • Social feedback integrated

Meso loops: 15-60 minutes

  • Group goals and challenges
  • Collaborative problem-solving
  • Shared victories

Macro loops: 60-180 minutes

  • Session commitment for all
  • Coordination possible
  • Stopping points respect group

Meta loops: Weeks to months

  • Relationship building over time
  • Campaign progression
  • Group identity formation

Pacing Pattern: Episodic

Episodic structure works best:

  • Clear session arcs
  • Complete experiences per session
  • Next episode scheduled together
  • Builds anticipation

Player Capacity Variants

Casual (30-60 minutes):

  • Micro: 5-15 seconds (simple coordination)
  • Meso: 15-30 minutes (short group challenges)
  • Macro: 30-60 minutes (single session = episode)
  • Example: Among Us, party games

Midcore (60-120 minutes):

  • Micro: 10-30 seconds (meaningful coordination)
  • Meso: 20-45 minutes (group missions)
  • Macro: 60-120 minutes (campaign session)
  • Example: It Takes Two, Deep Rock Galactic

Hardcore (2-4 hours):

  • Micro: 15-45 seconds (complex coordination)
  • Meso: 30-90 minutes (raid encounters)
  • Macro: 2-4 hours (raid night)
  • Example: Destiny 2 raids, Final Fantasy XIV raids

Examples

Example 1: It Takes Two (Midcore Fellowship)

Temporal architecture:

  • Micro: 10-20 seconds (cooperative actions)
  • Meso: 20-40 minutes (level/challenge)
  • Macro: 90-120 minutes (chapter)
  • Meta: 12-16 hours (full game)
  • Episodic structure (chapters)

Why it works:

  • Requires two players (pure Fellowship)
  • Continuous cooperation necessary
  • Clear chapter structure (stopping points)
  • Varied mechanics keep fresh
  • Perfect midcore session length

Measurables observed:

  • Cooperation required: Constant ✓
  • Session duration: 90-120 minutes ✓
  • Stopping points: Clear (chapters) ✓
  • Fellowship satisfaction: Very high ✓

Example 2: Among Us (Casual Fellowship)

Temporal architecture:

  • Micro: 5-15 seconds (actions + voting)
  • Meso: 8-12 minutes per round
  • Macro: 30-60 minutes (multiple rounds)
  • Meta: Ongoing (no progression)
  • Episodic (rounds)

Why it works:

  • Pure social deduction
  • Short rounds fit casual sessions
  • Drop-in drop-out friendly
  • Social interaction primary
  • No long-term commitment

Measurables observed:

  • Round duration: 8-12 minutes ✓
  • Casual session: 30-60 minutes ✓
  • Social engagement: Core experience ✓
  • Accessibility: Very high ✓

Example 3: Deep Rock Galactic (Midcore Fellowship)

Temporal architecture:

  • Micro: 10-20 seconds (combat + mining)
  • Meso: 20-35 minutes per mission
  • Macro: 60-120 minutes (multiple missions)
  • Meta: 50-200 hours (all classes, upgrades)
  • Episodic (missions)

Why it works:

  • 1-4 players, cooperation beneficial
  • Missions = clear episodes
  • Can drop-in mid-session somewhat
  • Class synergies encourage coordination
  • Good stopping points (mission end)

Measurables observed:

  • Mission duration: 20-35 minutes ✓
  • Cooperation: Required but flexible ✓
  • Stopping points: Every mission ✓
  • Fellowship: Strong when coordinating ✓

Common Mistakes

Mistake 1: No solo viable

  • Requires group to play at all
  • Hard to schedule
  • Blocks engagement
  • Solution: Solo mode or AI companions

Mistake 2: Poor coordination mechanics

  • Can't communicate effectively
  • No tools for planning
  • Synchronization too hard
  • Solution: Built-in comms, clear signaling

Mistake 3: Unequal time commitments

  • Some players need 10 hours, others 2
  • Creates social pressure
  • Resentment builds
  • Solution: Equal required investment

Mistake 4: No dropping in/out

  • Must commit entire session
  • Life happens, people leave
  • Group can't continue
  • Solution: Flexible group composition

Mistake 5: Poor session boundaries

  • No clear stopping points for group
  • "One more raid" trap
  • Social pressure to continue
  • Solution: Clear episodes, respect time

Integration Notes

Resources (Chapter 3):

  • Often shared resource pools
  • Team resource management
  • Scarcity creates cooperation needs
  • Abundance can reduce coordination

Cognitive Load (Chapter 4):

  • Communication adds load
  • Coordination = extra cognitive demand
  • Individual complexity can be moderate
  • Total load higher than solo games

Balance (Chapter 6):

  • Group balance critical
  • All players need meaningful roles
  • Can't have one dominant player
  • Difficulty scales with group size

Hybrid Aesthetics

Many games combine multiple aesthetics. Here's how temporal architecture adapts:

Challenge + Progression

Example: Hades, Dead Cells

Temporal pattern:

  • Challenge provides meso-loop structure (runs)
  • Progression provides meta-loop engagement (unlocks)
  • Micro loops: 5-15 seconds (combat)
  • Meso loops: 20-40 minutes (full run)
  • Macro loops: 60-90 minutes (multiple runs)
  • Meta loops: 20-80 hours (all unlocks)

Why combination works:

  • Challenge = immediate satisfaction
  • Progression = long-term engagement
  • Permanent upgrades ease difficulty curve
  • Both aesthetics served without conflict

Discovery + Narrative

Example: What Remains of Edith Finch, Firewatch

Temporal pattern:

  • Discovery drives exploration
  • Narrative provides emotional beats
  • Micro loops: 1-3 minutes (examine, read, listen)
  • Meso loops: 15-30 minutes (story vignettes)
  • Macro loops: 2-4 hours (complete story)
  • Meta loops: None (one playthrough)

Why combination works:

  • Discovery = player-driven pacing
  • Narrative = structured emotional arc
  • Both benefit from player-controlled timing
  • Natural synergy (discovering story)

Challenge + Fellowship

Example: Monster Hunter, Destiny 2 strikes

Temporal pattern:

  • Challenge = difficulty and mastery
  • Fellowship = cooperation and social
  • Micro loops: 10-20 seconds (combat)
  • Meso loops: 20-45 minutes (hunts/strikes)
  • Macro loops: 90-150 minutes (session)
  • Meta loops: 100-500+ hours (gear, mastery)

Why combination works:

  • Challenge creates coordination needs
  • Fellowship makes challenge social
  • Both benefit from episodic structure
  • Clear communication windows between combat

When Hybrids Fail

Submission + Tension:

  • Fundamentally opposed
  • Relaxation vs. pressure
  • Choose one as primary
  • Example of poor combination

Mastery + Casual Audience:

  • Capacity mismatch
  • Mastery requires time investment
  • Casual players lack capacity
  • Must simplify or retarget

When Standard Patterns Don't Apply

Experimental Temporal Structures

Time loops (Outer Wilds, Majora's Mask):

  • Fixed-duration meso loops (20 minutes, 72 minutes)
  • Discovery through repetition with variation
  • Player knowledge persists, world resets
  • Creates unique Discovery experience

Asynchronous multiplayer (Journey, Dark Souls messages):

  • Fellowship without synchronization
  • Temporal overlap but not simultaneous
  • Persistent world traces
  • Different temporal requirements

Real-time simulation (Animal Crossing, Forager):

  • Game time = real time
  • Can't binge progress
  • Daily engagement rhythm
  • Prevents burnout naturally

Roguelike deckbuilders (Slay the Spire, Monster Train):

  • Meta-progression within runs (deck building)
  • Each run unique
  • Hybrid temporal structure
  • Challenge + Mastery + Progression combined

Cultural and Regional Variations

Different cultures have different temporal preferences:

East Asian markets:

  • Often prefer shorter sessions
  • Mobile-first engagement
  • Gacha progression common
  • Session length: 5-15 minutes typical

Western markets:

  • Wide variance (casual to hardcore)
  • PC/console traditional
  • Session length: 30-120 minutes common

Mobile-first markets:

  • Very short sessions (5-10 minutes)
  • Pick-up-and-play essential
  • Progress in bite-sized chunks
  • Example: Idle games, merge games

Adjust temporal architecture for target market.

Conclusion

This catalog provides starting points, not rigid rules.

Remember:

  • Derive temporal requirements from YOUR aesthetic goals
  • Match temporal structure to YOUR player capacity
  • Validate with measurables specific to YOUR game
  • Iterate based on YOUR testing data

Use this catalog to:

  • Understand proven patterns
  • Avoid common mistakes
  • See how successful games implement temporals
  • Get starting values for your own derivations

Don't use this catalog to:

  • Copy numbers blindly
  • Skip derivation process
  • Ignore your specific context
  • Assume one-size-fits-all

The best temporal architecture serves your aesthetic goals while respecting your players' capacity.

These patterns emerged from analyzing hundreds of successful games. Learn from them, but design for your specific purpose.
