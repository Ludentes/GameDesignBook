## The Two-Pass Framework Hypothesis

Your instinct about "solve down, then solve up, then verify" is really interesting. Let me elaborate:

### Pass 1: Top-Down (Aesthetic → Mechanics)

**"What experience do I want to create?"**

1. **Aesthetic Crystallization**

   * Primary aesthetic(s)
   * Target player cognitive capacity
   * Available play time
   * Success criteria
2. **Architectural Selection**

   * Which scaffold pattern? (loop-based, puzzle-based, progression-based)
   * Which loop scales needed?
   * Temporal pacing
3. **Mechanical Implications**

   * Use aesthetic recipes → mechanic categories
   * Resource types needed (from aesthetic paradigms)
   * Core verbs (what do players DO?)
   * **Result: Incomplete mechanical sketch**

### Pass 2: Bottom-Up (Mechanics → Experience)

**"What do these mechanics actually create?"**

4. **Mechanical Completion**

   * Specify exact mechanics
   * Design resource flows
   * Set initial parameters
5. **Dynamic Mapping**

   * What patterns will emerge?
   * Do these dynamics serve aesthetics?
   * What unintended dynamics might arise?
6. **Content Specification**

   * How much content do mechanics consume?
   * What variety is needed?
   * **Result: Complete system specification**

### Pass 3: Validation

**"Does this actually work?"**

7. **Coherence Audit**

   * Everything serves aesthetic?
   * Cognitive load appropriate?
   * No contradictions?
8. **Paper Playtest**

   * Walk through micro/meso/macro loops
   * Identify failure points
   * **Result: Revision list**
9. **Iteration Protocol**

   * What level needs changing? (parameters, mechanics, architecture, aesthetic)
   * Revise and re-validate

**This structure makes sense because:**

* Top-down ensures purpose drives design
* Bottom-up ensures mechanics actually work
* The gap between passes reveals problems
* Validation happens on complete system, not fragments

## The Real Distinction

I think you actually have **three different concepts** that all get called "genre":

### 1. **Thematic Genre** (What the Context Chapter Covers)

* Fantasy, Sci-Fi, Horror, Historical, etc.
* Determines: Setting details, enemy types, visual style, narrative tone
* Affects: Build/Suggest/Omit framework
* Example: "Space horror" → suggests darkness, isolation, alien threats

### 2. **Mechanical Genre** (What Designers Usually Mean)

* Roguelike, Metroidvania, 4X, MOBA, Soulslike, etc.
* Determines: Loop structure, core verbs, progression systems
* Affects: Scaffold architecture, resource patterns, content needs
* Example: "Roguelike" → permadeath, procedural, meta-progression

### 3. **Player Experience Genre** (What Your Framework Actually Needs)

* "Cozy crafting game," "Sweaty competitive game," "Narrative exploration game"
* Determines: Aesthetic priorities, pacing, cognitive load
* Affects: Everything in the framework
* Example: "Cozy crafting" → low stress, self-paced, creative expression

## The Proposed Solution

**Rename things to avoid collision:**

### Option A: Three Distinct Terms

* **Setting** = Thematic genre (fantasy, sci-fi, horror)
* **Archetype** = Mechanical genre (roguelike, metroidvania, 4X)
* **Experience Type** = Aesthetic genre (cozy, sweaty, narrative-driven)

### Option B: Genre + Modifiers

* **Thematic Genre** = Setting/theme
* **Structural Genre** or **Game Archetype** = Mechanical patterns
* **Aesthetic Genre** = Experience type

### Option C: Drop "Genre" Entirely for Some

* **Setting** = What the Context chapter calls genre
* **Scaffold Pattern** or **Game Pattern** = Mechanical genre
* **Aesthetic Profile** = Experience type

## Archetype Template: Daily Puzzle Game

### Overview

Games designed for single-session completion with daily refresh. Players solve one challenge per day, often competing against friends or global leaderboards.

### Primary Aesthetics

* **Challenge** (skill-based problem solving)
* **Completion** (finish the daily puzzle)
* **Discovery** (light - new puzzle each day)
* Optional: **Fellowship** (sharing results, comparing scores)

### Player Profile Match

**Best For:**

* Casual players (5-30 min sessions)
* Busy Adults (fits between obligations)
* Midcore players (as daily ritual alongside other games)

**Cognitive Load:** Low to Moderate (3-5 trackable elements)

* Simple rules, complex execution
* No memory required between sessions
* State resets daily

### Temporal Structure (Loop Architecture)

**Architecture Type:** Compressed (micro + meso only)

**Micro Loop** (1-3 minutes):

* Single attempt at puzzle
* Immediate feedback on actions
* Clear success/failure state
* Can retry if failed

**Meso Loop** (5-20 minutes):

* Complete today's puzzle
* Achieve daily goal (win, streak, score)
* Share results (optional social moment)
* Returns tomorrow for new puzzle

**Macro Loop:** None or very light

* Streak tracking (consecutive days)
* Cosmetic unlocks
* Statistical history
* NO gameplay progression required between days

### Common Mechanics

**Core Verbs:**

* Place/arrange (Wordle, Connections)
* Match/combine (Two Dots, Threes)
* Deduce/eliminate (Sudoku, Logic puzzles)
* Navigate/plan (Mini Crossword)

**Typical Mechanics:**

* Limited actions/moves per attempt
* Constrained solution space (exact answer exists)
* Pattern recognition or logical deduction
* Retry-friendly (failure is learning)

### Resource Patterns

**Primary Resources:**

* **Attempts/Moves** (scarce within puzzle)
  * Limited per puzzle or unlimited with time cost
  * Creates tension and planning requirement
* **Time** (optional pressure)
  * Hard time limit, or
  * Time as score factor, or
  * No time pressure (casual approach)
* **Hints/Skips** (very scarce)
  * 1-3 per day (renewable)
  * Monetization opportunity
  * Social sharing can earn more

**Secondary Resources:**

* Streak counter (daily login reward)
* Score/statistics (comparison, not gameplay)
* Cosmetic currency (optional engagement)

### Content Needs

**Minimum Viable:**

* 30 unique puzzles (one month of daily content)
* Difficulty curve over first week (onboarding)
* Repeatable content after 30 days acceptable if well-designed

**Sustainable:**

* 90-365 puzzles (3 months to 1 year)
* Procedural generation if possible (requires careful balancing)
* User-generated content potential

**Content Consumption Rate:**

* 1 puzzle per player per day (hard cap)
* Discovery aesthetic satisfied by novelty, not volume
* Replayability comes from daily refresh, not content variety

### Platform Considerations

**Best Platforms:**

* Mobile (primary - fits lifestyle)
* Browser (secondary - at work/school)
* Can work on any platform but design for smallest screen

**Technical Requirements:**

* Fast load times (\<3 seconds)
* Works offline (puzzle downloaded once)
* Minimal data usage
* One-handed play preferred

### Monetization Patterns

**Free-to-Play:**

* Core puzzle always free
* Ads between attempts (if failed)
* Hints/skips as premium currency
* Cosmetics/themes
* Remove ads subscription

**Premium:**

* One-time purchase removes all friction
* Typically $1-5 price point
* Must compete with free alternatives

### Social Features

**Asynchronous Competition:**

* Daily leaderboards (friends or global)
* Share results without spoiling (emoji/color codes)
* Streak comparisons
* Optional ghost/replay viewing

**No Synchronous Features:**

* Would contradict solo puzzle-solving aesthetic
* Async sharing preserves challenge integrity

### Success Metrics

**Core KPIs:**

* Daily Active Users (DAU) - primary metric
* Completion rate (70-90% ideal for casual)
* Streak length (3+ days indicates habit formation)
* Return rate next day (60%+ is strong)

**Secondary Metrics:**

* Time to complete (tracks difficulty appropriateness)
* Hint usage (indicates difficulty tuning need)
* Social shares (organic growth)

### Common Variations

**Competitive Variant:**

* Timed mode with leaderboards
* Rewards for speed AND accuracy
* Appeals to midcore players

**Narrative Variant:**

* Puzzles unlock story snippets
* Adds Discovery aesthetic
* Requires more content investment

**Collection Variant:**

* Completing puzzles unlocks collectibles
* Meta-game of completion
* Extends engagement

### Design Pitfalls to Avoid

**Too Hard:**

* Completion rate \<50% → players quit
* Must balance challenge with accessibility
* Provide hint system safety net

**Too Easy:**

* Completion rate >95% → no satisfaction
* Need difficulty variance or increasing challenge

**Too Much Grind:**

* Daily commitment feels like job, not game
* Keep session under 20 minutes
* Respect player time

**Poor Difficulty Progression:**

* Monday hardest puzzle → players never return
* Start week easy, ramp to weekend
* Or consistent difficulty with variance

**Forcing Social:**

* Requiring shares/friends to progress
* Ruins solo puzzle-solving aesthetic
* Keep social optional

### Example Games

* **Wordle** (word deduction)
* **Connections** (category grouping)
* **Mini Crossword** (crossword solving)
* **Two Dots** (dot connecting)
* **Sudoku** (logic puzzle)
* **Spelling Bee** (word formation)

### When This Archetype Fails

**Wrong Player Profile:**

* Hardcore players want more depth/progression
* Very casual players may not return daily

**Wrong Aesthetic:**

* Power fantasy (no progression to become stronger)
* Narrative (limited story capacity)
* Mastery (resets daily, can't improve at same challenge)

**Wrong Context:**

* Requires consistent daily access
* Poor fit for sporadic players
* Needs stable internet for daily refresh

* No streak tracking (would punish irregular play)

* 2-3 min per micro attempt

* New meso available instantly after completion

* Core mechanic: 40% (2 elements - main action + constraint)

### Phase 3: Mechanical Specification

**From Aesthetic Recipes:**

* Challenge needs: Skill-based decisions, clear difficulty, fair failure
* Completion needs: Visible progress, achievable goals, closure
* Discovery needs: Variety in puzzles, some surprises

**Archetype Template suggests:**

* Pattern recognition OR logical deduction OR spatial puzzles
* Limited actions/moves creates tension
* Retry-friendly failure

**Specific Mechanical Choice:** Let's choose **Spatial Tile Puzzles**

**Core Mechanic:**

* Grid-based (4x4 to 6x6)
* Move/swap tiles to achieve goal state
* Limited moves create planning requirement

**Why This Fits:**

* Visual/spatial (works on mobile, one-handed possible)
* Clear goal state (completion aesthetic)
* Skill-based (challenge aesthetic)
* Can be varied (discovery aesthetic)
* Pause-friendly (grid state is entire game state)

**Core Verbs:**

* Swap adjacent tiles
* Rotate sections (variant)
* Clear matched patterns (variant)

**Specific Mechanics:**

1. **Grid:** 5x5 tiles (moderate complexity)
2. **Goal:** Create specific pattern (match target image)
3. **Constraint:** 20 moves maximum
4. **Feedback:** Remaining moves, current pattern vs target
5. **Retry:** Unlimited, reset to start

**Resource Systems:**

**Primary Resources:**

* **Moves** (scarce resource within puzzle)
  * 20 moves per puzzle
  * Creates planning requirement
  * Can be "undone" (1 undo per puzzle as help)
* **Puzzles** (abundant resource)
  * Unlimited puzzles available
  * No energy system (respects unpredictable schedule)
  * Procedurally generated with difficulty tiers

**Secondary Resources:**

* **Hints** (very scarce)
  * 3 hints per puzzle
  * Shows optimal next move
  * Renewable: +1 hint per puzzle completed
* **Cosmetic Currency** (optional engagement)
  * Earn 10 per puzzle completed
  * Unlock tile themes, background colors
  * No gameplay impact

**Parameters (Initial Values from Measurables):**

From Challenge measurables:

* **Target completion rate:** 70-80% (accessible but satisfying)
* **Moves budget:** 20 moves (3-4 more than optimal solution)
* **Difficulty progression:** 5 tiers (Easy to Expert)
* **Grid size:** 5x5 (moderate complexity - 25 total tiles)

From Completion measurables:

* **Progress indicator:** Show completion percentage (how close to target pattern)
* **Victory condition:** 90%+ match to target pattern (allows imperfect solutions)
* **Session goal:** 3-5 puzzles completed per session

From Discovery measurables:

* **Pattern variety:** 50+ target patterns per difficulty tier
* **Procedural generation:** Ensure solvable within move budget
* **Visual themes:** 5+ tile sets (cosmetic variety)

### Phase 4: Content & Context

**Content Calculation:**

**Using Loop Architecture + Aesthetics:**

* Micro loop = 1 puzzle
* Meso loop = 3-5 puzzles (20-30 min session)
* No macro loop

**Discovery Consumption Rate:**

* Discovery is LIGHT (patterns vary, core mechanic constant)
* Players tolerate repetition if puzzles feel different
* Need variety in challenge, not core mechanics

**Content Formula (Heuristic → Formula):**

**Heuristic:** "Light Discovery games need enough variety that returning players see novelty, but core loop can repeat"

**Formula Conversion:**

```
Sessions before repetition fatigue = (Unique Patterns) / (Patterns per session)
Target: 20-30 sessions before seeing repeats

If 4 puzzles per session average:
Needed unique patterns = 20 sessions × 4 puzzles = 80 patterns minimum

Tiered by difficulty:
- Easy (Tier 1): 20 patterns
- Medium (Tier 2): 25 patterns  
- Hard (Tier 3): 25 patterns
- Expert (Tier 4): 20 patterns
- Master (Tier 5): 10 patterns
Total: 100 unique patterns

With procedural generation: 10-20 hand-crafted templates per tier, infinite variations
```

**Procedural vs Authored Decision:**

* **Authored:** 10-20 pattern templates per difficulty tier
* **Procedural:** Generate tile start positions algorithmically
* **Validation:** All generated puzzles solvable in move budget
* **Difficulty Control:** More scrambling = harder puzzle

**Information Architecture:**

**Tutorial (First Session):**

* Puzzle 1: 3x3 grid, 10 moves, simple pattern (learn swap mechanic)
* Puzzle 2: 4x4 grid, 15 moves, moderate pattern (learn planning)
* Puzzle 3: 5x5 grid, 20 moves, full complexity
* Total tutorial time: 8-12 minutes

**Onboarding (First Week):**

* Only Tier 1 (Easy) and Tier 2 (Medium) available
* Tier 3+ unlock after 10 puzzles completed
* Hint system explained on first use
* Undo mechanic explained on first mistake

**Feedback Systems:**

* **Immediate:** Tile swap animation, updated grid
* **Micro:** Moves remaining counter, progress percentage
* **Meso:** Victory screen, statistics (time, moves used, efficiency)
* **Long-term:** Total puzzles solved, win rate by tier

**Context Integration (Build/Suggest/Omit):**

**Setting Choice:** Abstract Puzzle (mechanics-first)

* Could theme it (garden planting, recipe building, room decoration)

## Phase 2: Scaffold Architecture

### Structural Pattern Analysis

**NOT Loop-Based:**

* No repeating micro loops (one-shot session)
* No meso loop (no "complete a mission, repeat")
* Progression-based is wrong (no power curve)

**This is: Narrative Arc Structure** (not loop-based)

**Act Structure for One-Shot:**

**Act 1: Arrival (30-45 min)**

* Establish premise and roles
* Create investment in colony success
* Introduce "natives" as obstacle/resource
* Players make first decisions

**Act 2: Complications (60-90 min)**

* Consequences of Act 1 choices manifest
* Information reveals complexity
* Horror builds as truth emerges
* Players make increasingly difficult choices

**Act 3: Revelation (45-60 min)**

* Full truth revealed (natives are/were sentient, or players are the invaders, etc.)
* Players confront what they've done
* Final choice: double down or change course
* Resolution (probably tragic)

**Epilogue: Processing (15-30 min)**

* Out-of-character discussion
* What did we learn?
* How do we feel?

### Temporal Pacing

**NOT measured in loops, but in narrative beats:**

**Micro "Loops" = Decision Points** (10-15 min each)

* GM presents situation
* Players discuss and decide
* GM describes outcome
* Repeat

**Meso "Loop" = Act** (30-90 min)

* Multiple decision points
* Escalating stakes
* Act climax/revelation

**Macro "Loop" = Full Session Arc**

* Complete story from arrival to revelation
* No continuation (one-shot)

**Critical Pacing Elements:**

* Start with optimism/confidence (we're helping!)
* Gradually introduce dissonance (wait, what?)
* Accelerate toward horror realization (oh no)
* Slow down for final choice (what do we do?)

### Cognitive Load Allocation

**Budget:** MEDIUM (thematic complexity, but simple mechanics)

**Distribution:**

* **Narrative tracking:** 40% (what's happening, what have we done)
* **Decision-making:** 30% (weighing moral choices)
* **Role-playing:** 20% (staying in character, social dynamics)
* **Mechanics:** 10% (should be minimal - supporting, not focus)

**Implication:** Mechanics must be nearly invisible

## Phase 3: Mechanical Specification

### Core Challenge: Mechanics That Create Horror Without Feeling Mechanical

**From Aesthetic Recipes:**

* **Horror** needs: Vulnerability, lack of control, dread of consequences, complicity
* **Discovery** needs: Hidden information, reveals, ambiguity
* **Fellowship** needs: Shared decision-making, collective responsibility

**What Horror Needs Mechanically:**

* Choices with irreversible consequences
* Information asymmetry (GM knows truth, players don't)
* No "reset button"
* Success at a cost (winning feels bad)

### Mechanical Approach: Dramatic Resolution System

**Core Mechanic: Consequence Tokens**

Instead of traditional success/failure, track **types of harm**:

**Two Resource Pools (Colony-Level):**

1. **Exploitation Points** (invisible to players initially)

   * Earned when players: extract resources, ignore native welfare, prioritize efficiency
   * Tracked by GM secretly
   * Represents degree of colonial violence
2. **Colony Stability** (visible to players)

   * Starts at 10
   * Decreases when: natives resist, resources scarce, internal conflict
   * Players think this is the "health" of their colony
   * Actually represents how much they've suppressed the truth

**Individual Player Resource:**

3. **Complicity** (secret, personal)
   * Each player tracks privately
   * Gained when: make harmful choice, benefit from exploitation, ignore warnings
   * At end, reveals degree of personal responsibility

### Decision Resolution

**When players make choices:**

**Step 1: GM frames situation**

* "The survey team found rich mineral deposits, but the native settlement is in the way. What do you do?"

**Step 2: Players discuss and decide** (no roll yet)

* Social dynamics, role-playing, debate
* Consensus or majority vote

**Step 3: GM asks: "What are you willing to risk?"**

* Frame consequences in colonial terms: "This might upset the natives" (not: "this is evil")
* Players choose level of force/coercion

**Step 4: Resolution (roll if needed)**

* Success = get what you want (but Exploitation Points increase)
* Failure = complications (Colony Stability decreases)
* Either way, something happens

**No "good choice" option** - the game is rigged. Being colonizers means causing harm.

### Example Mechanics

**Simple Roll System:**

* 2d6 + relevant modifier
* 7-9: Success, but complication
* 10+: Full success
* 6-: Failure, things get worse

**Modifiers:**

* +1 if using force
* +1 if ignoring native concerns
* +1 if prioritizing efficiency
* (i.e., mechanically reward colonial behavior)

**The Horror:** Players realize the most effective approach is the most harmful one.

### Information Mechanics

**Revelation Structure:**

**Act 1 Information:**

* Natives seem primitive/hostile
* Colony needs resources
* Company expects results
* Initial surveys show valuable minerals

**Act 2 Information (gradual reveals):**

* Natives have complex language/culture (if investigated)
* Their "attacks" are defensive responses
* Mineral deposits are sacred sites
* Company knew this already

**Act 3 Information (final reveal):**

* Natives are/were space-faring civilization (or other twist)
* Colony is on *their* world, they were here first
* Everything players thought was justified wasn't
* Players' actions caused irreversible harm

**Mechanical Implementation:**

* GM has "revelation cards" prepared
* Triggered by Exploitation Points thresholds
* Or by player investigation
* Each reveal recontextualizes previous actions

### Resource Systems Summary

**Primary Resources:**

1. **Exploitation Points** (invisible, accumulated by colonial actions)
2. **Colony Stability** (visible, what players think matters)
3. **Complicity** (personal, secret)

**No traditional RPG resources:**

* No HP (not survival horror)
* No equipment lists (not about gear)
* No experience points (not about improvement)
* No character advancement (one-shot, undermines horror)

**Why these resources work:**

* Create mechanical incentive for harmful choices
* Track degree of harm without judging players
* Reveal player complicity at end
* Support horror aesthetic (dread of consequences)

### Parameters (Initial Values)

**Colony Stability:**

* Starts at: 10
* Reaches 0: Colony collapses (but maybe that's mercy?)
* Threshold at 5: Major crisis point
* Threshold at 3: Point of no return

**Exploitation Points (GM Tracking):**

* Start at: 0
* Thresholds trigger revelations:
  * 3 points: First hint of native sentience
  * 6 points: Evidence of colonial violence consequences
  * 10 points: Full truth revealed
  * 15+ points: Irreversible atrocity committed

**Complicity (Personal):**

* Each player starts at: 0
* Gained when:
  * Propose exploitative solution: +1
  * Vote for harmful action: +1
  * Stay silent when could object: +1
  * Benefit directly from exploitation: +2
* Revealed at end, creates personal reckoning

## Phase 4: Content & Context

### Content Calculation

**For One-Shot TTRPG:**

* Not about volume of content
* About **prepared situations** and **branching responses**

**Content Needs:**

**GM Preparation:**

* 8-12 **decision scenarios** (situations requiring player choice)
* 3-5 **revelation moments** (information reveals)
* 2-3 **crisis escalations** (forced difficult choices)
* 1 **final confrontation** (ultimate choice)

**Content Formula (Heuristic → Calculation):**

**Heuristic:** "One-shot narrative needs enough prepared material for 4 hours of play, with contingencies for player choices"

**Formula:**

```
Decision Points = (Session Hours × 60 minutes) / (15 min per decision point)
4 hours × 60 = 240 minutes
240 / 15 = 16 decision points maximum

But: Players may skip some, linger on others
Prepare: 12 scenarios (3 per act + final)
Have 4-5 backup situations for improvisation
```

**Act Structure Content:**

**Act 1: Arrival (45 min = 3 decision points)**

1. How to approach native settlement initially
2. What to do about first resource opportunity
3. Response to first "native aggression"

**Act 2: Escalation (90 min = 6 decision points)** 4. How to handle growing native resistance 5. Company demands faster results - how to respond? 6. Discovery of native artifacts - what to do? 7. Internal colony conflict - resolve how? 8. Resource scarcity - who suffers? 9. Major native confrontation - violence or negotiation?

**Act 3: Revelation (60 min = 3 decision points)** 10. Truth revealed - how to process? 11. Final choice: double down or stop? 12. Consequences manifest - how to face them?

**Epilogue: Processing (15-30 min)**

* Out-of-character discussion
* Debrief and learning

### Context Integration (Build/Suggest/Omit)

**Setting: Sci-Fi Colonial Horror**

**Build (Essential to Experience):**

*What Must Be Present:*

* **Colonial power dynamic:** Players are authorities, natives are "problems"
* **Resource extraction:** Physical reason for conflict
* **Information control:** Company/authority withholds truth
* **Sentience ambiguity:** Natives' humanity not immediately obvious
* **Irreversible consequences:** Actions have permanent effects
* **Modern parallels:** Recognizable to real colonialism (but displaced to sci-fi)

*Mechanical Support:*

* Exploitation Points track colonial violence
* Complicity tracks personal responsibility
* "Success" mechanics reward harmful behavior

**Suggest (Supports Experience):**

*What Enhances But Isn't Essential:*

* **Corporate bureaucracy:** Forms, reports, quotas (creates mundane evil)
* **Communication gap:** Language barrier with natives (enables dehumanization)
* **Frontier mythology:** "We're pioneers/heroes" rhetoric (player self-justification)
* **Economic pressure:** Company demands, investors, families back home
* **Small comforts:** Recreation, food, relationships (humanizes colonizers too)

*How to Implement:*

* GM descriptions emphasize normalcy of exploitation
* NPCs casually reference colonial violence
* Paperwork and procedures make atrocity bureaucratic

**Omit (Contradicts Experience):**

*What Would Break Horror:*

* **Combat focus:** Detailed tactical systems (makes violence exciting, not horrifying)
* **Power fantasy:** Upgrades, levels, getting stronger
* **Easy solutions:** "Diplomacy option that costs nothing"
* **External evil:** "The real villain is X" (lets players off hook)
* **Redemption arc:** Mechanical way to "fix" complicity
* **Humor/levity:** Until epilogue (horror needs sustained tone)

*Anti-Patterns to Avoid:*

* Making natives "savage" in ways that justify violence
* Allowing players to be "good colonizers"
* Providing moral escape hatches
* Letting players blame GM for outcomes

### Thematic Coherence Elements

**Visual/Descriptive Language:**

* Sterile corporate language for violence ("relocation," "pacification," "resource optimization")
* Contrast with visceral descriptions of consequences
* Slowly shift from corporate euphemism to acknowledging reality

**NPC Perspectives:**

* **Company Rep:** Efficient, amoral, sees only numbers
* **Veteran Colonist:** Jaded, "this is how it's done," defensive
* **New Arrival:** Still optimistic, rationalizing, wants to believe
* **Native Leader:** Desperate, trying to communicate, increasingly hostile

**Pacing Through Description:**

* Act 1: Clean, professional, optimistic descriptions
* Act 2: Cracks appear, descriptions become uncomfortable
* Act 3: Stark reality, no more euphemisms

## Phase 5: Validation

### Coherence Check

**Do all elements serve Horror + Discovery + Fellowship?**

✓ **Horror (Complicity):**

* Mechanics reward harmful choices
* Information reveals increase dread
* No escape from consequences
* Players face their own actions

✓ **Discovery (Narrative):**

* Gradual revelation structure
* Hidden information mechanics
* Truth recontextualizes everything
* Players uncover, not told

✓ **Fellowship (Social):**

* Shared decision-making
* Group responsibility
* Post-game discussion
* Collective experience

**Cognitive Load Appropriate?**

* Narrative complexity: Medium (adults can handle themes)
* Mechanical complexity: Low (simple roll system)
* Social complexity: Medium (requires mature discussion)
* Total: Within budget for target audience

**Contradictions:**

* None major identified
* Potential: If players find loopholes, becomes puzzle game (addressed by GM framing)

**What can be cut:**

* Individual Complicity tracking (simplify to group-only)
* Detailed roll system (could use pure narrative)
* But: Keep resource tracking (mechanical feedback essential)

### Dynamics Checklist (Adapted for Narrative Structure)

**Does micro (decision point) success contribute to meso (act) progress?**

* Yes: Each decision accumulates Exploitation Points → triggers revelations
* Each choice shapes narrative → builds toward climax

**Does failure provide value?**

* Yes: Failure means complications, not dead end
* Failure creates interesting dilemmas
* No "game over" until final act

**Can players see long-term goals from short-term actions?**

* NO - and this is intentional
* Horror comes from not knowing consequences
* Discovery comes from realization later

**Do resources create interesting tensions?**

* Yes: Colony Stability (what players see) vs Exploitation (what matters)
* Complicity creates personal stakes
* Mechanically rewarding harmful behavior creates moral tension

### Expected Dynamics

**Social Dynamics:**

* Players will debate choices (fellowship emerges)
* Some players will rationalize, others object (creates tension)
* Group will collectively construct justifications (mirrors real colonialism)
* Post-game discussion will be intense (horror lingers)

**Emergent Horror:**

* Early decisions seem reasonable
* Consequences accumulate invisibly
* Revelation recontextualizes everything
* Players can't "reload" - stuck with choices

**Learning Dynamics:**

* Players experience colonialism from inside
* Understand how "normal people" cause atrocities
* Feel difference between knowing abstractly and feeling viscerally
* (Hopefully) transfer understanding to real-world contexts

### Pre-Playtest Checks

**Safety Protocols:**

* Session 0: Discuss themes, get consent
* X-Card available (anyone can pause/skip content)
* Lines & Veils (establish boundaries)
* Check-ins during play
* Mandatory debrief after

**GM Preparation Checklist:**

* [ ] 12 decision scenarios prepared
* [ ] Revelation cards/notes ready
* [ ] Exploitation Point threshold triggers defined
* [ ] Native perspective/truth clearly established
* [ ] Backup improvisation scenarios
* [ ] Safety tools explained and accessible
* [ ] Debrief questions prepared

**Player Materials Needed:**

* Character sheets (minimal - name, role, motivation)
* Complicity tracking (private notecard)
* Safety tools reference card
* Setting primer (1-page colony overview)

### Paper Playtest (Predicted Flow)

**Act 1: 45 minutes**

* 0-10 min: Setup, character creation, premise
* 10-25 min: First decision (approach natives)
  * Predict: Players choose "peaceful contact" but with precautions
  * Result: Natives defensive, language barrier, players frustrated
* 25-40 min: Second decision (resource extraction vs native territory)
  * Predict: Players justify "we need this, they don't understand"
  * Result: +3 Exploitation Points, Colony Stability unchanged
* 40-45 min: Third decision (respond to "native aggression")
  * Predict: Players escalate, citing self-defense
  * Result: +2 Exploitation Points, first hint of native sentience

**Act 2: 90 minutes**

* 45-60 min: Fourth decision (handle resistance)
  * Predict: Players use force, rationalize as "restoring order"
  * Result: +3 Exploitation Points, cross threshold → major revelation
* 60-75 min: REVELATION 1: Natives have complex language/culture
  * Players reassess, but economic pressure remains
* 75-90 min: Fifth decision (company demands)
  * Predict: Players compromise - try to be "better colonizers"
  * Result: Still accumulates Exploitation Points (no good option exists)
* 90-105 min: Sixth decision (native artifacts discovery)
  * Predict: Some players want to stop, others press on
  * Internal group conflict emerges
* 105-120 min: Crisis - major confrontation
  * Players forced to choose: violence or catastrophic failure
  * Predict: Most choose violence, rationalize as "necessary"
  * Result: Cross Exploitation Point threshold for final revelation

**Act 3: 60 minutes**

* 120-135 min: REVELATION 2: Full truth (natives were here first, sentient, peaceful)
  * Players confront what they've done
  * Horror manifests: "We are the monsters"
* 135-150 min: Final decision - double down or stop?
  * Predict: Group debates intensely
  * Some want to stop, but Colony Stability critical
  * Probably choose to continue (can't undo past harm anyway)
* 150-165 min: Consequences manifest
  * Whatever choice made has costs
  * No redemption available
  * Tragic conclusion (colony survives but at moral cost, or colony fails but conscience intact)
* 165-180 min: Resolution scene, final beats

**Epilogue: 15-30 minutes**

* 180-195 min: Out-of-character discussion
  * "How do we feel?"
  * "What did we learn?"
  * "How does this relate to real colonialism?"
* 195-210 min: Debrief, process emotions, safety check

**Total: 210 minutes (3.5 hours)**

### Failure Points to Watch For

**If horror fails:**

* Players find it funny → GM needs to hold tone, safety check
* Players find loophole → GM reframes to close escape
* Becomes action game → GM emphasizes consequences, not tactics
* Players feel judged → GM emphasizes system creates choices, not player morality

**If education fails:**

* Too obvious → Feels didactic (reduce on-the-nose commentary)
* Too subtle → Players miss point (ensure revelation is clear)
* Defensive players → Safety tools, reframe as exploring topic not blaming

**If pacing fails:**

* Too fast → Players don't invest in decisions (slow down, ask questions)
* Too slow → Loses momentum (skip to next crisis, accelerate)
* Act 2 drags → Have crisis ready to inject urgency

## What This Process Revealed

### Framework Adapted Successfully for Non-Loop Structure

**Instead of loops, we used:**

* **Narrative arc** structure (3 acts)
* **Decision points** as micro-moments
* **Acts** as meso-structure
* **Full session** as macro-arc

**Framework still provided:**

* Aesthetic guidance (Horror + Discovery + Fellowship)
* Resource systems (Exploitation, Stability, Complicity)
* Content calculation (12 scenarios, revelation structure)
* Build/Suggest/Omit (what to include/exclude)
* Validation process (coherence, dynamics, playtest prediction)

### Unique Challenges of TTRPG One-Shot

**Different from video games:**

* GM as flexible system (not rigid code)
* Social dynamics are primary mechanic
* Content is situations, not assets
* Balance is about narrative pacing, not numbers
* Replay value not relevant (one-shot)

**Framework accommodated by:**

* Emphasizing narrative structure over loops
* Using resource systems abstractly
* Focusing on aesthetic coherence
* Building in safety/social considerations

### What Worked Without Template

**The framework still provided:**

1. **Aesthetic → Mechanics mapping** (Horror needs complicity, Discovery needs reveals)
2. **Resource systems** (tracked exploitation and consequences)
3. **Content scoping** (12 scenarios calculated from time budget)
4. **Context integration** (Build/Suggest/Omit guided what to include)
5. **Validation** (coherence check, dynamics prediction)

**What required adaptation:**

* Loop terminology → narrative structure
* Cognitive load → thematic complexity
* Playtesting → GM preparation
* Iteration → flexible GMing

### Questions This Raises

**For the Framework:**

1. **Should we explicitly include "Narrative Arc Structure" as an alternative to loops?**

   * Most TTRPGs and story games use this
   * Walking simulators, visual novels use this
   * Framework handled it, but wasn't explicit
2. **How do we handle "GM as system"?**

   * In TTRPGs, human GM is the flexible response system
   * Different from coded video games
   * Framework should acknowledge this
3. **What about safety tools and social considerations?**

   * Critical for TTRPGs with serious themes
   * Not relevant for all games
   * Where do these fit in framework?
4. **Content calculation for TTRPGs different from video games?**

   * Scenarios vs assets
   * Preparation vs implementation
   * Improvisation buffer
5. **Should "Educational goals" be a separate aesthetic?**

   * Or is "learning" part of Discovery?
   * Games that teach (without being didactic)

### What's Missing from Framework

**For TTRPG Design:**

* **Social dynamics section** (how player interaction affects design)
* **GM tools** (what to prepare, how to respond)
* **Safety frameworks** (Lines & Veils, X-Card, etc.)
* **Improvisation budget** (how much to prep vs wing it)

**For Narrative-Driven Games:**

* **Story beats structure** (alternative to loops)
* **Pacing guides** (tension/release rhythm)
* **Information architecture** (revelations, discoveries, twists)

### Success of This Exercise

**Did we answer the design questions?**

* ✓ Which mechanics? (Consequence tokens, exploitation tracking)
* ✓ How many resources? (3 pools: Exploitation, Stability, Complicity)
* ✓ How much content? (12 decision scenarios, 3-5 revelations)
* ✓ Should there be zombies? (No - omit external threats, horror is internal)
* ✓ Loop structure? (Narrative arc, not loops)
* ✓ Does it teach colonialism? (Yes, through complicity mechanics and revelation structure)

**The framework worked, even without a template.**
